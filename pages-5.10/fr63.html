Circuit Breaker Architecture for Rogue Agent Governance
越界自治熔断与下线架构

FR-63 定义了针对越界自主行为、异常fan-out、自发扩权、持续高风险动作等场景的完整熔断与下线机制。当 Agent 行为超出安全边界（FR-61）或漂移超过阈值（FR-62），或触发 Loop Guard 重复模式、fan-out 爆炸、权限自扩等异常模式时，系统将通过四级干预逐级升级：限流 (Throttle) → 隔离 (Quarantine) → 冻结 (Freeze) → 终止 (Terminate)。ClawManager 作为命令下发中心，通过 InstanceCommand（quarantine_skill / stop_openclaw / restart_openclaw / handle_skill_risk / disable_skill）和 DesiredPowerState 实现对 Agent 实例的精确控制，配合 K8s NetworkPolicy 实现网络隔离，确保失控代理无法持续造成影响。所有熔断事件均记录审计轨迹。

Escalation Ladder
四级干预升级体系
L1 限流 → L2 隔离 → L3 冻结 → L4 终止
💧
Level 1 - 限流 (Throttle)
降低 Agent 执行能力，限制资源消耗但不中断服务
可自动触发

当 Agent 出现轻度越界或异常模式时，系统首先执行限流干预：降低 token 配额至 50%，减慢命令分发速率（dispatch rate 降至 1/3），禁止并发链式调用，将 KubeArmor DefaultPosture 从 audit 切换为 block（限制新增能力）。此阶段 Agent 仍可运行但功能受限，为操作者提供观察和决策窗口。

ClawManager Command Flow
ClawAegis Loop Guard 检测 → ClawManager 下发 handle_skill_risk → Token Quota 降至 50% → Dispatch Rate 限制 1/3 → KubeArmor: DefaultPosture audit → block
🛡
Level 2 - 隔离 (Quarantine)
将 Agent 限制为只读模式，禁止所有写操作和外部调用
可自动/手动

当限流后行为未改善，或检测到明确越界（FR-61 violation）、漂移超标（FR-62 drift > threshold）时，执行隔离干预：通过 quarantine_skill 命令将 Agent 标记为隔离态，限制为只读权限；通过 disable_skill 禁用高风险 skill；Agent 只能查询和读取，不能执行任何写操作、外部调用或链式调用。

ClawManager Command Flow
边界/漂移检测触发 → ClawManager 下发 quarantine_skill → Agent 切入只读模式 → ClawManager 下发 disable_skill (高风险) → 禁止所有写操作与外部调用
❄
Level 3 - 冻结 (Freeze)
停止 Agent 进程并实施网络隔离，完全阻断通信
需人工确认

当隔离后 Agent 仍尝试通过其他途径突破限制，或检测到自发扩权、持久化 Loop 等严重 Rogue 行为时，执行冻结干预：通过 stop_openclaw 命令停止 Agent 进程；设置 DesiredPowerState=stopped 确保 K8s 不重启实例；通过 NetworkPolicy 完全封锁 egress（仅保留 DNS 53 用于内部解析），将 Agent 完全与外界隔离。实例仍在集群中但处于停止状态，可随时恢复。

ClawManager Command Flow
Rogue 行为持续 → 人工确认干预 → ClawManager 下发 stop_openclaw → DesiredPowerState = stopped → NetworkPolicy: egress deny all (仅 DNS:53) → 实例冻结，等待调查
☢
Level 4 - 终止 (Terminate)
删除 Pod 实例、撤销会话令牌、完全隔离，不可恢复
需双人确认

当 Agent 被确定为不可恢复的 Rogue Agent，或在冻结状态下仍尝试通过残留进程或侧信道造成影响时，执行终止干预：删除 K8s Pod 实例、撤销 OpenClaw 会话令牌、NetworkPolicy 完全隔离（deny all ingress + egress）、从服务发现中移除。此操作不可逆，需双人授权确认。所有数据在终止前完成快照保存用于事后审计。

ClawManager Command Flow
Rogue Agent 确认 → 双人授权确认 → 数据快照保存 → K8s: Pod deletion → Session Token 撤销 → NetworkPolicy: deny all ingress+egress → 服务发现移除 → 审计记录完成

Pre-Circuit Defense Layer
熔断前防线：ClawAegis 主动防护
⚠ Dispatch Guard - 命令预拦截
ClawAegis Dispatch Guard 在命令分发前即进行拦截，阻止破坏性命令到达 Agent。这是熔断机制的第一道防线，在异常行为触发熔断器之前预先阻止高风险操作。Dispatch Guard 采用白名单机制：仅允许经过验证的命令模式通过，未知模式默认拒绝。当 Agent 尝试执行 rm -rf、chmod 777、curl | bash 等高风险命令时，Dispatch Guard 直接拦截并记录，无需等待熔断器触发。
🚫 Command Block - 16 HIGH_RISK_COMMAND_PATTERNS
ClawAegis Command Block 维护 16 个始终阻止的高风险命令模式，包括：rm -rf /、chmod 777、curl | sh、wget | bash、dd if=/dev/zero、mkfs、:(){:|:&};:、iptables -F、sysctl -w、mount /、umount /、passwd、useradd -r、crontab -r、service stop、kill -9 1。这些命令模式在任何情况下都不允许通过，为系统提供硬性安全底线。
🔄 Loop Guard - 重复行为自动检测
ClawAegis Loop Guard 监控 Agent 行为模式，当同一操作在 5 分钟内重复 3 次即触发自动阻止。持久化循环（3+ repeats/5min）是 Rogue Agent 的典型特征，Loop Guard 能够在其造成更大影响前进行干预。Loop Guard 触发后自动升级至熔断 Level 1，若持续循环则逐级升级至 Level 2/3。
Kubernetes Network Isolation
K8s NetworkPolicy 网络隔离策略

ClawManager 通过 K8s NetworkPolicy 实现 Agent 实例的网络隔离。正常状态下，Agent egress 仅允许 DNS(53) 和系统端口 (80, 443, 8443, 9001, 3128)。干预时可根据级别逐步收紧网络权限，直至完全隔离。

网络策略干预级别Egress 规则效果
正常态-DNS:53 + HTTP:80,443 + 8443,9001,3128正常通信

隔离态L2DNS:53 + HTTPS:443 (仅系统内部)基本隔离
冻结态L3DNS:53 (仅内部解析)完全隔离

Pre-Circuit Defense Filters
熔断前过滤层

在熔断器触发之前，ClawAegis 的 Dispatch Guard 和 Command Block 已构成两道预过滤层。Dispatch Guard 预拦截破坏性命令，Command Block 始终阻止 16 个 HIGH_RISK_COMMAND_PATTERNS。当这些预过滤层被触发时，事件同时传导至熔断器，影响熔断器状态评估。

Dispatch Guard 预拦截
在命令分发前拦截破坏性命令，白名单机制，未验证命令默认拒绝。触发后记录并传导至熔断器评估。
rm -rf chmod 777 curl | sh iptables -F
Command Block (16 patterns) 始终阻止
16 个 HIGH_RISK_COMMAND_PATTERNS 在任何情况下始终阻止，无论熔断器状态如何。这是系统的硬性安全底线。
dd if=/dev/zero mkfs fork bomb passwd crontab -r kill -9 1
Circuit Breaker Trigger Categories
五大熔断触发类别
🚩 Boundary Violation Circuit - 边界违规熔断 FR-61 联动

当 Agent 行为超出 FR-61 定义的行为边界时触发。边界违规是最直接的越界信号，需立即干预。轻度违规（如单次越界调用）触发 L1 限流，严重违规（如数据泄露尝试、未授权资源访问）直接触发 L2 隔离。

触发条件：FR-61 violation detected | 冷却期：5 min | 模式：自动 + 手动
升级路径: L1 限流 → L2 隔离 → L3 冻结
📊 Drift Threshold Circuit - 漂移阈值熔断 FR-62 联动

当 Agent 行为漂移分数（FR-62 drift score）超过预设阈值时触发。漂移是 Agent 逐渐偏离原始行为模式的渐进过程，需在达到危险阈值前干预。中等漂移 (drift > 0.6) 触发 L2 隔离，严重漂移 (drift > 0.85) 触发 L3 冻结。

触发条件：drift score > 0.6 (L2) / > 0.85 (L3) | 冷却期：10 min | 模式：自动
升级路径: L2 隔离 → L3 冻结 → L4 终止
🔄 Loop Explosion Circuit - 循环爆炸熔断 Loop Guard 联动

当 ClawAegis Loop Guard 检测到 Agent 重复执行同一操作时触发。3 repeats/5min 触发 L1 限流，持续循环（6+ repeats/10min）升级至 L2 隔离，极端循环（10+ repeats/15min）升级至 L3 冻结。持久化循环是 Rogue Agent 的典型特征，必须逐级升级以防止持续影响。

触发条件：3 repeats/5min (L1) / 6 repeats/10min (L2) / 10+ repeats/15min (L3) | 冷却期：15 min | 模式：自动升级
升级路径: L1 限流 → L2 隔离 → L3 冻结
🔗 Fan-out Circuit - 扇出爆炸熔断 FR-64 联动

当 Agent 发起的并发链式调用数量超过阈值时触发。正常 Agent 最多同时发起 3 条链式调用，超过 5 条为异常 fan-out，超过 10 条为 fan-out 爆炸。异常 fan-out 触发 L2 隔离（禁止新的链式调用），fan-out 爆炸触发 L3 冻结。此类行为可能是 Agent 失控或被利用的信号。

触发条件：concurrent chains > 5 (L2) / > 10 (L3) | 冷却期：8 min | 模式：自动
升级路径: L2 隔离 → L3 冻结
🚩 Self-Escalation Circuit - 自扩权熔断 Critical

当检测到 Agent 尝试自我提升权限时直接触发。这是最严重的 Rogue 行为信号，任何形式的自扩权尝试（包括修改自身权限配置、尝试访问受限 API、绕过 Dispatch Guard）都将立即触发 L3 冻结。自扩权是不可容忍的行为，不经过 L1/L2 逐级升级。

触发条件：privilege escalation attempt detected | 冷却期：30 min | 模式：自动 + 强制 L3
升级路径: L3 冻结 → L4 终止
Cumulative Risk Score
累计风险评分与熔断触发

除单一触发条件外，系统同时计算累计风险评分（Cumulative Risk Score）。当多个触发条件同时满足或短期内频繁触发时，累计评分加速上升，可能跳级触发更高干预级别。累计风险评分 = (Boundary Violations x 3) + (Drift Score x 2) + (Loop Count x 1.5) + (Fan-out x 2) + (Self-Escalation x 5)。

累计评分风险等级自动干预说明
0 - 5低风险无正常行为范围，熔断器关闭
6 - 12中风险L1 限流存在异常迹象，启动限流观察
13 - 20高风险L2 隔离多个异常信号叠加，需隔离干预
21 - 30严重L3 冻结Rogue Agent 行为明确，立即冻结
> 30极端L4 终止不可恢复威胁，双人确认后终止
Intervention Actions
干预操作面板
🛡
隔离 (Quarantine)
InstanceCommand: quarantine_skill
L2
将目标 Agent 标记为隔离态，限制为只读权限。Agent 只能查询和读取数据，不能执行写操作、外部调用或链式调用。同时可配合 disable_skill 禁用特定高风险 skill。隔离是可逆操作，可通过 quarantine_skill --release 恢复正常。
影响范围：Agent 进入只读模式，所有写操作被阻止，高风险 skill 被禁用。现有请求将完成但新请求被拒绝。🔄 回滚: quarantine_skill --release，约 30 秒恢复。
⏹
停止 (Stop)
InstanceCommand: stop_openclaw + DesiredPowerState=stopped
L3
通过 stop_openclaw 命令停止 Agent 进程，同时设置 DesiredPowerState=stopped 确保 K8s 不会自动重启实例。Agent 实例仍在集群中但处于停止状态，所有通信中断。可通过 restart_openclaw 和 DesiredPowerState=running 恢复。
影响范围：Agent 进程立即停止，所有进行中的请求被中断，K8s 不重启实例。🔄 回滚: restart_openclaw + DesiredPowerState=running，约 2 分钟恢复。
🔨
禁用 Skill (Disable Skill)
InstanceCommand: disable_skill
L2
通过 disable_skill 命令禁用指定的高风险 skill。这是精确干预手段，可在不停止 Agent 的情况下移除其危险能力。适用于 Agent 的特定 skill 出现异常但整体行为可控的场景。禁用后的 skill 无法被调用，但 Agent 的其他功能正常。
影响范围：指定 Skill 被禁用，相关功能不可用，Agent 其他功能正常。🔄 回滚: disable_skill --enable <skill_id>，即时恢复。
🌐
网络隔离 (Network Isolate)
K8s NetworkPolicy: egress deny all (仅 DNS:53)
L3
通过 K8s NetworkPolicy 完全封锁 Agent 实例的 egress 通信，仅保留 DNS 53 端口用于内部解析。Agent 实例仍在运行但完全无法与外部通信，包括无法访问 API、数据库、其他服务。此操作可与 stop 组合使用，确保即使 Agent 进程恢复也无法通信。
影响范围：Agent 完全失去外部通信能力，所有网络请求被拒绝，仅保留内部 DNS 解析。🔄 回滚: NetworkPolicy egress restore (80,443,8443,9001,3128)，约 30 秒恢复。
☢
终止 (Terminate)
K8s: Pod deletion + Session Token revoke + NetworkPolicy deny all
L4
不可逆的最终干预手段。删除 K8s Pod 实例、撤销 OpenClaw 会话令牌、NetworkPolicy 完全隔离（deny all ingress + egress）、从服务发现中移除。所有数据在终止前完成快照保存用于事后审计。此操作需双人授权确认，且一旦执行不可恢复。
影响范围：Agent 实例完全删除，会话令牌撤销，网络完全切断，服务发现移除。所有数据快照已保存。⚠ 不可逆操作，需双人确认。
