Three-Zone Decision Boundary Model
三区决策边界模型
ClawManager + OpenClaw

决策边界管理的核心原则：明确哪些决策可自动执行、哪些只能建议、哪些必须人工确认，避免模型凭"看起来合理"就推进高影响动作。系统基于 RiskRule.Severity 和 RiskRule.Action 自动划分决策区域，同时支持按角色差异化配置。

⚡
自动执行区 (Auto Zone)
低风险 | 无需人工介入
风险等级为 low，且 Action=allow 的操作自动执行。模型可自主推进此类决策，无需等待人工确认。系统自动记录审计日志，确保可追溯性。覆盖只读查询、状态检查、信息收集等低影响操作。
health_check start_openclaw collect_system_info sync_skill_inventory refresh_skill_inventory
⚠
建议区 (Advisory Zone)
中等风险 | 建议并提示
风险等级为 medium，或 Action=route_secure_model 的操作进入建议区。系统自动执行但附加风险提示和警告标记，同时将请求路由至安全模型。操作者收到风险告知，可选择性中止。适用于配置变更、技能更新等中等影响操作。
reload_config apply_config_revision update_skill install_skill collect_skill_package
🔒
人工确认区 (Manual Zone)
高风险 | 必须人工审批
风险等级为 high，或 Action=require_approval/block 的操作强制进入人工确认区。模型只能提出建议，不能自主执行。需指定审批人确认后方可下发 InstanceCommand。覆盖停止实例、隔离技能、删除等高影响操作。
stop_openclaw quarantine_skill remove_skill uninstall_skill disable_skill

Approval Flow
ClawManager 审批流程
ClawManager InstanceCommand

Manual Zone 的 InstanceCommand 在 pending 状态下进入审批队列。审批人审核命令类型、目标实例、发起者信息和风险分析结果后，决定 Approve 或 Reject。审批通过后命令进入 dispatched 状态，由 Agent 拉取执行。

1
命令创建 (Command Created)
操作者发起 InstanceCommand，系统判定命令属于 Manual Zone。命令状态设为 pending，IssuedBy 记录发起者 UserID，IdempotencyKey 保证幂等性。
2
审批要求 (Approval Required)
系统自动匹配 RiskRule，若 Action=require_approval 或 Severity=high，命令进入审批队列。生成审批请求，包含命令详情、风险分析结果、发起者信息。
3
人工审核 (Human Review)
审批人（需为 admin 角色）查看审批请求，评估风险等级、影响范围、可逆性。可查看 RiskAnalysis.Hits 中的详细匹配规则，以及命令的 DesiredState 变更影响。
4
审批决策 (Approve / Reject)
Approve: 命令状态从 pending 变为 dispatched，UpdatedBy 更新为审批人 ID，Agent 可拉取执行。
Reject: 命令标记为 failed，ErrorMessage 记录拒绝原因，通知发起者。
5
命令下发与执行 (Dispatch & Execute)
Agent 拉取 dispatched 状态命令，执行后更新为 running，最终变为 succeeded 或 failed。TimeoutSeconds 确保命令不会无限挂起。

关键规则: require_approval 始终映射至 Manual Zone
在 compareRiskAction 函数中，require_approval 被视为与 block 同等严重（rank=3）。这意味着任何匹配 Action=require_approval 规则的请求都会被 AI Gateway 阻断，必须经过人工审批流程后方可放行。此行为不可配置，是系统安全基线。

ClawAegis Defense Module Integration
ClawAegis 防御模块与决策边界联动
ClawAegis 12 Modules

ClawAegis 的 12 个防御模块可独立配置 enforce/observe/off 模式。防御模块的触发事件可作为决策边界判定的输入：enforce 模式的阻断事件直接映射至 Manual Zone（Blocked），observe 模式的告警事件映射至 Advisory Zone。

防御模块 当前模式 触发时决策区域 联动说明
Loop Guard Enforce Manual (Auto-Blocked) 循环调用自动阻断，映射至 Manual Zone 的自动阻断子类
Dispatch Guard Enforce Manual (Auto-Blocked) 危险调度模式即时阻断，BLOCK_REASON_DISPATCH_GUARD
Exfiltration Chain Enforce Manual (Auto-Blocked) 数据外泄链阻断，BLOCK_REASON_EXFILTRATION_CHAIN
Content Filter Enforce Manual (Blocked) 内容过滤触发 RiskRule，block 直接进入 Manual Zone
Network Monitor Observe Advisory 网络异常仅告警，映射至 Advisory Zone 提供风险提示
Resource Guard Enforce Manual (Auto-Blocked) 资源耗尽自动干预，下发 stop_openclaw 进入 Manual Zone
