各阶段日志来源与记录方式
阶段描述日志来源系统核心组件记录内容关联键
Input 用户请求输入 ClawManager AI Gateway SessionID, RequestID, UserID, InstanceID, 原始 Prompt trace_id, session_id

Recovery 修复恢复动作 KSecure ClawManager SecLog + Command 处置动作, 恢复结果, 验证状态, KSecure SecLog 恢复记录 trace_id, host_name
AI Observatory Architecture
ClawManager AI Observatory 审计服务架构
运行中
🏢 AI Observatory 核心能力
✓ 55KB 审计逻辑 — 覆盖全生命周期事件的完整审计逻辑
✓ 审计事件服务 — Audit Event Service 统一事件入口
✓ ModelInvocation 完整记录 — 请求/响应/Token/Cost/Latency
✓ CostRecord 关联 — 每个决策关联成本记录
✓ InstanceCommand 生命周期 — 4阶段时间戳 + 结果JSON
✓ InstanceConfigRevision 校验 — SHA256 完整性
✓ gateway.risk.detected 事件 — 风险检测审计
🛡️ ClawAegis 防御事件日志
✓ before_dispatch Hook — 动作分发前审计记录
✓ before_agent_reply Hook — Agent 回复前审计记录
✓ 审批流程日志 — 人工审批动作与结果
✓ 风险拦截日志 — 被阻断的操作记录
✓ 审批人记录 — 审批人身份与审批时间
✓ 审批超时记录 — 审批超时自动处理
✓ 与 AI Observatory 联动 — trace_id 关联
✅ 验收标准
✓ 安全事件有完整日志 — 8阶段全覆盖，从 Input 到 Recovery 每个环节都有对应系统记录
✓ 日志包含时间/来源/目标/动作 — 每条日志必须包含 Timestamp, Source, Target, Action 四要素
✓ 支持按时间/类型/实例查询 — 多维度查询支持，时间范围 + 事件类型 + 实例ID 组合过滤
Log Specification
生命周期日志规范

每个生命阶段的日志必须包含: 时间戳(Timestamp)、来源(Source)、目标(Target)、动作(Action) 四要素。跨系统日志通过 trace_id 或 PodName+Timestamp 关联。
