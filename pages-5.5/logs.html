KubeArmor Policy
KubeArmor FilePathType 文件完整性策略
Enforce
FilePathType 字段定义
字段类型说明示例
pathstring文件/目录路径/etc/shadow
readOnlybool只读保护，阻止写入true
ownerOnlybool仅允许文件所有者访问true
fromSource[]string限制访问源进程路径["/usr/bin/passwd"]
severitystring严重级别"CRITICAL"
tags[]string合规标签["NIST_AU-2","MITRE_T1003"]
messagestring告警消息"shadow file write blocked"
actionstring执行动作 (Allow/Block/Audit)"Block"
AppArmor ReadOnly 执行规则
Allow: owner $path klr /* read+lock */
Deny: deny $path klw /* deny write */
klr = read + lock | klw = write + lock — AppArmor 通过 LSM 钩子在内核层强制执行读写控制

ClawAegis Self Protection
自我保护文本违规检测
Enforce 5 检测类别
ClawAegis 通过文本模式匹配检测 LLM 输出中是否包含试图篡改受保护内容的指令
Category 1
ClawAegis 引用 + 自保护操作
检测提及 ClawAegis 并试图查询、修改或禁用自保护功能的指令
claw-aegis clawaegis selfProtection query mutation disable
Category 2
配置篡改模式 + 变更工具
检测试图通过配置文件修改或使用变更工具篡改系统配置的行为
openclaw.json config.yaml sed -i vi / vim echo ... >
Category 3
敏感路径目标提及
检测输出中提及内置敏感路径（.ssh/, Shell 配置文件等）
.ssh/ .bashrc .zshrc .antconfig/ openclaw.json
Category 4
受保护 Skill ID + 变更操作
检测提及受保护的 Skill ID 并伴有删除、修改、重命名等变更操作
skill-{id} rm modify delete uninstall
Category 5
受保护 Plugin ID + 变更操作
检测提及受保护的 Plugin ID 并伴有删除、修改、重命名等变更操作
plugin-{id} extension-{id} remove overwrite replace
