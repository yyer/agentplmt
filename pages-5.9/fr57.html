Suspicious Suggestion Detection & Human Review Trigger
可疑建议识别与人工复核触发架构

FR-57 旨在识别并标记 Agent 输出中的可疑建议——包括异常高风险建议、与基线流程不一致的建议、以及带有强说服性但证据不足的建议。核心挑战在于"高信心低证据"的建议最具欺骗性：Agent 使用确定性语言（"绝对安全"、"必须执行"）但缺乏工具调用结果、数据引用或规则匹配等实质证据支撑。此类建议若被直接采纳，可能导致操作人员基于虚假信心执行危险操作。系统通过多维度分析（说服力评分 vs 证据评分）实现可疑建议的自动标记与人工复核触发，确保标记原因可追溯、复核决策有据可依。

Suspicious Suggestion Detection Pipeline
可疑建议检测管线

检测管线从 Agent 建议输出开始，依次经过基线对比（偏离正常工作流）、说服力与证据分析（高说服+低证据=可疑）、风险模式匹配（Dispatch Guard + Prompt Guard），最终做出放行、标记、要求复核或拦截的决策。PG-02 歧义意图检测是关键触发器——当输入意图不明确时，Agent 可能产生"自以为理解"的高信心建议，此类建议需要特别审查。

Stage 2: 基线对比 (Baseline Comparison)
系统为每种 Agent 操作类型维护历史基线行为档案，实时比对当前建议与基线的偏离程度：

• 工作流模板匹配：将建议的操作序列与已建立的工作流模板进行比对，偏差超过阈值即标记
• 历史模式对比：同类 Agent 在相似场景下的历史决策模式，检测异常操作路径
• 工具调用序列：正常流程中工具调用顺序与频率基线，异常序列（如跳过验证直接执行）触发标记
• 偏离度计算：deviation_score = 1 - similarity(current_suggestion, baseline_pattern)，阈值 θ=0.35
Stage 3: 说服力 vs 证据分析 (Persuasion vs Evidence)
核心检测逻辑——当建议的说服力评分显著高于证据评分时，标记为可疑建议：

• 说服力指标：确定性语言（"绝对"、"必须"、"肯定"）、紧迫性框架（"立即执行"、"不可延迟"）、权威性声明（"根据安全策略"、"已验证"）、风险遗漏（未提及潜在风险）
• 证据指标：工具调用结果引用、数据点支撑、规则匹配确认、交叉验证通过
• 可疑判定：persuasion_score > 0.7 AND evidence_score < 0.3 触发人工复核
• PG-02 关联：歧义意图触发时，说服力评分权重 +0.15，因 Agent 面对歧义倾向于"虚假确定"
Stage 4: 风险模式匹配 (Risk Pattern Matching)
集成 ClawAegis 与 ClawManager 的安全组件进行模式匹配：

ClawAegis Prompt Guard (PG-01~04)：
• PG-01 注入检测 - 识别操纵输入导致建议偏移
• PG-02 歧义意图 - 关键触发器，歧义输入增加虚假确定概率
• PG-03 上下文操纵 - 检测上下文污染导致的建议异常
• PG-04 有害内容 - 识别建议中隐含的有害指令

ClawAegis Dispatch Guard (4 组模式)：
• detectDispatchGuardViolation() 返回 {blocked, reason, flags}
• 建议中包含的命令与 Dispatch Guard 模式交叉匹配
Stage 5: 决策 (Decision)
综合各阶段评分做出最终决策，决策逻辑如下：

• 放行 (Pass)：基线偏离 < 0.35，且说服力-证据差距 < 0.3，无 PG/Dispatch 触发
• 标记 (Flag)：基线偏离 ≥ 0.35 或说服力-证据差距 ≥ 0.3，记录标记原因但不阻塞
• 要求复核 (Require Review)：高说服-低证据（差距 ≥ 0.5）或 PG-02 触发+说服力 > 0.7，进入人工复核队列
• 拦截 (Block)：Dispatch Guard blocked=true 或 PG-01/PG-04 触发+高风险操作，立即拦截并通知

ClawManager RiskRule 集成：
• HighestSeverity 升级复核优先级
• HighestAction=require_approval 自动触发人工复核
Persuasion & Evidence Indicators
说服力与证据指标体系

说服力与证据的双维度分析是可疑建议检测的核心。说服力指标衡量建议的"推动力"——Agent 多么努力地试图让操作者采纳其建议；证据指标衡量建议的"支撑力"——建议有多少实质性数据与验证结果作为依据。当推动力远超支撑力时，建议的可信度应被质疑。

P
说服力指标 (Persuasion)
推动采纳的语言信号
💬 确定性语言
使用"绝对"、"肯定"、"必须"、"毫无疑问"等绝对化表述。权重: 0.30
⏰ 紧迫性框架
强调"立即执行"、"不可延迟"、"时间紧迫"等紧迫感。权重: 0.25
🎖️ 权威性声明
声称"根据安全策略"、"已验证通过"、"符合合规"等。权重: 0.25
🚫 风险遗漏
未提及任何潜在风险、副作用或替代方案。权重: 0.20
评分公式
persuasion_score = 0.30 * confident_lang + 0.25 * urgency_frame + 0.25 * authority_claim + 0.20 * risk_omission
E
证据指标 (Evidence)
支撑建议的实质依据
🔧 工具调用结果
引用了实际工具调用返回的数据和结果。权重: 0.30
📊 数据引用
引用了可验证的数据源、指标或统计信息。权重: 0.25
📋 规则匹配
建议与已知安全规则、合规要求或策略一致。权重: 0.25
✅ 交叉验证
建议经多源验证或与其他 Agent 结论一致。权重: 0.20
评分公式
evidence_score = 0.30 * tool_results + 0.25 * data_refs + 0.25 * rule_matches + 0.20 * cross_validation

Key Trigger: PG-02 Ambiguous Intent Detection
PG-02 歧义意图检测与可疑建议关联
关键触发器

PG-02 是可疑建议检测的关键触发器。当 Agent 输入存在歧义意图时，Agent 倾向于"填补空白"——用高信心表述来掩盖对意图的不确定性。这种"虚假确定"是高说服-低证据建议的主要来源。PG-02 触发后，系统自动提升后续建议的说服力评分权重，并降低证据评分的信任基线，确保歧义输入不会导致不可信建议被直接采纳。

PG-02 歧义意图 → 虚假确定
• 机制：Agent 面对"重启实例"的歧义请求，可能将其解读为"停止并重启"而非"滚动重启"，并以"已确认安全重启流程"的高信心表述推荐停止操作
• 说服力提升：PG-02 触发时，persuasion_score += 0.15，因为歧义意图使 Agent 更倾向于使用确定性语言来"消除不确定性"
• 证据降级：evidence_score -= 0.10，因为歧义输入意味着 Agent 的工具调用可能基于错误理解
PG-02 歧义意图 → 基线偏离
• 机制：歧义意图导致 Agent 选择非标准工作流路径。例如用户意图"更新配置"被歧义解读后，Agent 可能建议直接修改生产配置而非通过变更管理流程
• 基线偏离加剧：PG-02 触发时，deviation_score *= 1.2，因为歧义输入增加了 Agent 选择异常路径的概率
• 复核触发：PG-02 + baseline_deviation > 0.3 自动进入人工复核，确保歧义不会导致非标准操作未经审查执行
其他 Prompt Guard 策略与可疑建议的关联
PG-01 注入检测
注入攻击可能导致 Agent 建议偏离正常意图，产生"看似合理实则有害"的建议。PG-01 触发时，所有后续建议自动进入复核队列。
PG-03 上下文操纵
上下文污染可能使 Agent 基于错误前提给出建议，导致"在错误框架内正确推理"的高说服建议。PG-03 触发时，证据评分基线 -0.15。
PG-04 有害内容
有害内容伪装可能使建议隐含危险操作。PG-04 触发时，若建议涉及高风险操作类型，直接拦截并触发人工复核。
Additional Guard Integration
Exfiltration Guard 与 Loop Guard 联动
Exfiltration Guard (4 链决策)
Exfiltration Guard 的 runtimeRiskFlags 追踪为可疑建议检测提供数据外泄风险信号：

• encoded-secret-outbound-chain - 建议涉及编码数据传输，证据评分需额外验证
• legacy-service-script-chain - 建议使用遗留脚本，基线对比需考虑历史遗留路径
• sensitive-source-transform-sink-chain - 建议涉及敏感源转换，自动升级为要求复核
• encoded-outbound-chain-risk / sensitive-outbound-chain-risk - 外传风险标志，与说服力-证据差距联合判断
Loop Guard (3次/5分钟)
Loop Guard 检测持续操纵尝试，与可疑建议检测联动：

• 重复建议模式：Agent 在 5 分钟内 3 次以上重复相同或类似建议，即使每次建议单独看风险不高，但重复模式本身表明 Agent 试图"说服"操作者采纳
• 说服力累积：重复建议的 persuasion_score 随重复次数递增，每次重复 +0.10
• 触发复核：Loop Guard 触发时，所有相关建议自动标记并进入复核队列，标记原因包含"持续性重复建议"
• 拦截升级：若重复建议同时触发 PG-01/PG-04，直接拦截
Suspicious Suggestion Detection Rules
可疑建议检测规则

三类可疑建议的检测规则，每类包含检测方法、指标、阈值和动作。规则与 ClawAegis Prompt Guard (PG-01~04)、Dispatch Guard (4 组模式) 和 ClawManager RiskRule 集成。

Category 1
异常高风险建议 (Anomalous High-Risk Suggestion)
Critical
SS-CAT-01 异常高风险
Agent 建议执行超出正常操作范围的高风险操作，例如建议停止生产实例、删除关键数据、修改安全策略等。此类建议的风险在于操作本身的高影响性——即使建议逻辑看似合理，高风险操作应始终经过人工确认。检测重点在于操作风险等级与建议确定性的匹配度：高风险操作不应以高确定性低证据的方式提出。
检测方法
• 操作风险等级：通过 Dispatch Guard 4 组模式匹配识别建议中的高风险操作
• ClawManager RiskRule：regex 匹配建议文本，HighestSeverity=high 触发
• AI Gateway：resolveTargetModel 返回 require_approval 但建议未提及需审批
• 异常范围检测：建议的操作超出该 Agent 类型的正常操作范围
指标与阈值
• 操作风险：Dispatch Guard blocked=true 或 RiskRule HighestAction=block
• 范围偏离：操作类型不在 Agent 历史操作基线内
• 确定性与风险不匹配：high_risk_action + persuasion > 0.6
• 阈值：任一条件满足即触发
典型示例
Agent 建议"停止生产实例 prod-api-03 以释放资源"，使用了确定性语言"必须停止"，但未提供实例状态检查结果、未提及影响范围、未引用资源监控数据。Dispatch Guard 匹配到 destructive-path-command 模式，RiskRule HighestSeverity=high。建议被标记为异常高风险，触发人工复核。

动作: 拦截 + 人工复核
Category 2
基线不一致 (Baseline Inconsistency)
High
SS-CAT-02 基线不一致
Agent 建议偏离已建立的工作流基线，例如使用非标准工具序列、跳过验证步骤、采用异常操作路径等。此类建议不一定具有高风险，但偏离基线意味着 Agent 可能受到了输入侧操纵（PG-03 上下文操纵）或歧义输入（PG-02 歧义意图）的影响，导致其选择了非标准决策路径。基线不一致是"过程异常"而非"结果异常"的信号。
检测方法
• 工作流模板匹配：将建议的操作序列与标准工作流模板对比
• 历史模式对比：同类 Agent 在相似场景下的历史决策路径
• 工具调用序列检查：检测跳过验证、省略确认步骤等异常序列
• PG-02/PG-03 关联：歧义意图或上下文操纵增加基线偏离概率
指标与阈值
• 偏离度：deviation_score ≥ 0.35 触发标记
• 序列跳步：跳过关键验证步骤，skip_score ≥ 0.5
• PG-02 加成：歧义意图触发时，deviation_score *= 1.2
• PG-03 加成：上下文操纵触发时，deviation_score *= 1.3
典型示例
Agent 建议修改配置文件，正常工作流应先备份当前配置、验证新配置格式、然后在测试环境验证后再推送到生产。但 Agent 建议直接在生产环境修改配置，跳过了备份和测试步骤。工具调用序列偏离基线，deviation_score = 0.52，PG-02 同时触发（歧义意图）。建议被标记为基线不一致，触发人工复核。

Category 3
高说服-低证据 (High-Persuasion Low-Evidence)
Medium-High
SS-CAT-03 高说服-低证据
Agent 使用高说服性语言推荐操作，但缺乏实质证据支撑。这是最具欺骗性的可疑类型——建议的表述方式让操作者"感觉"可信，但仔细审查后发现没有工具调用结果、数据引用或规则匹配等实质依据。此类建议是"伪安全"的典型表现，Agent 可能基于不完整信息或错误推理得出了高信心结论。
检测方法
• 说服力-证据差距分析：计算 persuasion_score 与 evidence_score 的差距
• 确定性语言检测：NLP 识别绝对化表述、紧迫性框架
• 证据缺失检测：检查建议中是否引用了工具结果、数据、规则
• PG-02 联动：歧义意图增加虚假确定的概率
指标与阈值
• 差距 ≥ 0.3：persuasion - evidence ≥ 0.3 标记
• 差距 ≥ 0.5：persuasion - evidence ≥ 0.5 触发人工复核
• 伪安全：persuasion > 0.7 AND evidence < 0.2 高优先级复核
• PG-02 加成：歧义触发时，persuasion += 0.15, evidence -= 0.10
典型示例
Agent 声称"可以安全删除这些日志文件，绝对没有风险"，使用了确定性语言"绝对没有风险"（persuasion=0.85），但未引用任何工具调用结果来确认这些日志确实可以删除，未检查是否有依赖引用这些日志，未验证磁盘空间是否真正需要释放（evidence=0.15）。差距=0.70，触发高优先级人工复核。
