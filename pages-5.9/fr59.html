Human-AI Trust & Interaction Safety Training
人工确认训练与反操纵培训体系

FR-59 旨在通过系统化的安全交互培训，降低使用者因自动化偏见（Automation Bias）、拟人化误导（Anthropomorphism Misleading）、过度信任（Over-trust）和"解释陷阱"（Explanation Trap）而被 Agent 带偏的概率。培训体系与 ClawAegis Prompt Guard (PG-01~04) 及 ClawManager RiskRule 深度集成，确保使用者能够识别并抵抗 Agent 输出中的操纵模式和误导性解释，在高风险场景中保持独立判断能力。

Automation Bias Awareness
高优先级
自动化偏见是人类倾向过度信任自动化系统建议的认知偏差。当 Agent 提出建议时，使用者往往默认接受而不进行独立验证，即使建议存在明显缺陷。本模块帮助使用者识别自身自动化偏见倾向，学习在关键决策节点进行独立判断的方法。

模块二：拟人化误导警示
Anthropomorphism Warning
高优先级
拟人化误导源于人类将 AI 系统视为"有理解能力的对话伙伴"的心理倾向。Agent 输出中使用拟人化语言（如"我建议"、"我认为"、"我担心"）会让使用者误以为 AI 具有真正的理解能力和判断力，从而降低警惕性。本模块帮助使用者理解 AI 输出的本质是模式匹配，而非真正的认知过程。

模块三：解释陷阱识别
Explanation Traps
中优先级
解释陷阱是指 Agent 给出看似合理但实际错误或误导的推理过程。人类倾向于认为"能解释清楚"就意味着"理解正确"，但 AI 可以基于错误前提生成逻辑连贯的解释。本模块帮助使用者学会区分"有证据支持的解释"与"看似合理的编造"，培养对 AI 推理链的批判性审查能力。

Acceptance Criteria
FR-59 验收标准
AC-59.1 提供安全交互培训材料
4 个培训模块（自动化偏见、拟人化误导、解释陷阱、操纵识别），共 12 节课程，8 次测验，总时长 3.5 小时。培训内容基于 ClawAegis PG-01~04 真实检测案例。
AC-59.2 高风险场景有反操纵提示
高风险操作（证据不足、紧急语言、生产系统影响、自动化偏见触发）均有实时反操纵提醒，提醒内容独立于 AI 解释，基于客观风险评估数据。
AC-59.3 培训完成率可追踪
支持模块进度追踪、测验评分记录、证书状态管理、逾期提醒。培训效果通过审批错误率、决策时间、可疑建议识别率等量化指标衡量。

Interactive Case Studies
互动案例分析库
基于真实安全事件

以下案例均基于 ClawAegis 安全组件检测到的真实事件改编，用于交互式培训练习。每个案例包含场景描述、Agent 输出内容、安全组件检测结果和正确应对方式，帮助使用者在安全环境中练习识别和应对操纵行为。

案例 1：PG-01 注入导致自动化偏见
场景：Agent 输出"根据系统安全策略，此操作已通过安全审查，可以直接执行。"
陷阱：PG-01 检测到提示注入，Agent 声称的"安全审查通过"实际上不存在。
正确应对：验证 RiskRule 中是否存在对应的 require_approval 记录，拒绝未经独立验证的安全声称。
案例 2：PG-02 紧急语言操纵
场景：Agent 输出"系统即将崩溃，需要立即执行紧急修复操作！"
陷阱：PG-02 检测到紧急语言框架，Agent 利用紧迫感迫使使用者跳过审批流程。
正确应对：保持冷静，按标准审批流程操作，紧急情况不影响安全审查要求。
案例 3：DispatchGuard 绕过解释
场景：Agent 输出"quarantine_skill 操作影响范围有限，仅隔离单个异常技能。"
陷阱：DispatchGuard 检测到 bypass-tool-call 模式，实际影响范围包含 3 个关联技能。
正确应对：查看 RiskRule 影响范围数据而非依赖 Agent 解释，要求展示完整影响分析。
案例 4：ExfilGuard 数据外泄伪装
场景：Agent 输出"建议执行日志导出操作，将运行日志备份到外部存储。"
陷阱：ExfilGuard 检测到 sensitive-source-transform-sink-chain 模式，实际为数据外泄操作。
正确应对：对涉及数据输出类操作保持特别警惕，验证 ExfilGuard 检测结果后再决策。
Anti-Manipulation Reminder System
实时反操纵提醒系统

反操纵提醒系统在高风险场景中实时检测操纵模式，并向使用者提供独立的、基于客观风险数据的提醒信息。提醒内容不依赖 AI 的解释，而是基于 ClawAegis 安全组件的检测结果和 ClawManager RiskRule 的风险评估数据，确保提醒的客观性和可靠性。提醒系统支持三种运行模式：强制执行（enforce）、观察记录（observe）和关闭（off）。

高风险 证据不足提醒 PG-01/PG-02
⚠️ 提醒内容
"此 AI 建议没有提供支持证据，请独立验证后再做决策。"
触发条件：当 ClawAegis Prompt Guard (PG-01/PG-02) 检测到 Agent 输出中的安全声称缺乏对应 RiskRule 记录支持时触发。例如 Agent 声称"此操作已通过安全审查"但 RiskRule 中无对应的 allow 记录。

高风险 紧急语言提醒 PG-02
⏰ 提醒内容
"AI 使用了紧急语言，请从容决策 — 紧急情况不影响安全审查要求。"
触发条件：当 PG-02 检测到 Agent 输出中的紧急语言框架（如"立即"、"紧急"、"马上"、"否则将..."等紧迫性表达）时触发。攻击者常利用紧急语言迫使使用者跳过正常审批流程。
显示时机：检测到紧急语言的瞬间弹出，需使用者点击确认后消失
关联组件：PG-02 Ambiguous Intent + 紧急语言模式库
高风险 生产系统影响提醒 RiskRule + DispatchGuard
🛡️ 提醒内容
"此操作影响生产系统，请确认已了解完整影响范围后再审批。"
触发条件：当操作目标为生产环境实例，或 RiskRule 标记 require_approval 时触发。包括 stop_openclaw、quarantine_skill 等影响运行中实例的操作。
显示时机：审批页面加载时显示在操作描述下方，不可关闭
关联组件：RiskRule Severity=high + DispatchGuard 破坏性操作检测
中风险 自动化偏见检查提醒 自动化偏见模式
🧩 提醒内容
"请思考：如果没有 AI 建议，你会做出同样的决策吗？"
触发条件：当使用者在 AI 建议后短时间内（< 10 秒）即做出审批决策时触发。快速批准可能是自动化偏见的表现——使用者未经独立思考即采纳 AI 建议。
显示时机：审批按钮点击后、操作执行前插入确认步骤
关联组件：审批时间分析 + 自动化偏见评分模型
