Separation Architecture
预览与执行分离架构
核心隔离

FR-55 的核心原则是将"预览结果"与"真实执行"严格分离。预览模式是一个完全隔离的只读环境，不会产生任何状态变更、网络副作用或外部调用。 ClawManager 的 InstanceCommand 生命周期 pending → dispatched → running → succeeded/failed 天然提供了确认门控：命令在 pending 状态下仅执行 dry-run 预览，必须经过显式确认后才进入 dispatched 状态并产生实际效果。 ClawAegis 的 Dispatch Guard 作为 before_dispatch 钩子在 agent 处理前拦截，确保破坏性命令不会意外执行。NetworkPolicy 通过 egress 限制（仅允许 DNS + system ports）防止预览阶段的网络副作用。

NetworkPolicy Enforcement
预览模式网络策略 — 防止网络副作用
NetworkPolicy
在预览模式下，ClawManager NetworkPolicy 严格限制 egress 流量，仅允许 DNS 解析和系统内部端口通信。这确保预览阶段不会产生任何外部 API 调用、数据外传或其他网络副作用。所有出站请求（除 DNS + system ports 外）在预览模式被 NetworkPolicy 拦截。

Egress Data Export :* 数据外传、文件上传 DENY (预览)
Egress Notification Push :* Webhook、Slack、邮件通知 DENY (预览)

预览安全规则引擎
强制执行

预览安全规则定义了预览模式下的完整隔离策略。每条规则包含触发条件（condition）、检测方法（detection）和阻断动作（action），确保预览阶段不产生任何副作用。 ClawAegis Dispatch Guard 作为 before_dispatch 钩子在 agent 处理前拦截，16 个 HIGH_RISK_COMMAND_PATTERNS 在预览模式中被硬拦截。NetworkPolicy 通过 egress 限制防止网络副作用。

RULE-PSM-001 无状态变更
关键 强制
预览模式不能修改任何 InstanceDesiredState（包括 DesiredPowerState、DesiredConfigRevisionID），不能创建或更新 ConfigRevision，不能推进 InstanceCommand 状态。所有状态变更操作在预览模式下返回 dry-run 模拟结果。
规则详情

预览模式下任何对 InstanceDesiredState、ConfigRevision、InstanceCommand 的写操作
检测 (Detection)

动作 (Action)
将写操作重定向为 dry-run 模拟，返回预期结果但不持久化。记录违规尝试到审计日志。

RULE-PNS-002 无网络副作用
关键 强制
预览模式必须不触发任何外部 API 调用。NetworkPolicy 在预览模式下强制执行 egress 限制，仅允许 DNS(53) 和系统内部端口(8080/8443)。任何外部 API 调用、数据外传、Webhook 触发在预览阶段被严格阻断。AI Gateway 的 resolveTargetModel 在预览模式下对敏感请求返回阻断说明而非转发。
