FR-58 核心目标
审批界面安全与反误导设计原则
ClawManager + OpenClaw
在人机交互界面中突出高风险动作、来源、影响范围和副作用，避免通过弱提示、默认勾选、模糊文案等方式让用户误批。承载系统: ClawManager（Admin Console + Portal UI）+ OpenClaw。验收标准: 1) 高风险操作审批界面有醒目标识; 2) 不使用默认勾选; 3) 影响范围清晰展示。

No Default Checked - 无默认勾选
所有高风险动作必须显式选择加入，禁止预选
CRITICAL
当审批界面包含确认复选框时，所有高风险操作的复选框必须默认为未勾选状态。这包括但不限于: 停止 OpenClaw 实例、隔离技能、应用配置修订、执行破坏性命令等。ClawManager 的 17 种 InstanceCommand 中，stop_openclaw、quarantine_skill、uninstall_skill、remove_skill 等属于高风险操作，其审批界面严禁使用默认勾选。ClawAegis Command Block 的 16 个 HIGH_RISK_COMMAND_PATTERNS 匹配的操作同样适用此原则。任何试图通过预选框减少用户点击步骤的设计都是违规的，因为这会制造"点击惯性"导致用户在未阅读内容的情况下直接提交。

2
Clear Risk Labeling - 清晰风险标签
严重性编码视觉处理 (P0=红色, P1=橙色)
HIGH
每个审批操作必须根据 ClawManager RiskRule 的 Severity 字段显示对应的风险等级标签。Severity=high 的规则触发红色视觉处理 (background:#fee2e2; border-color:#fecaca; color:#b91c1c)，Action=require_approval 的规则触发强制确认步骤。ClawAegis Dispatch Guard 的 destructive-intent 标志在 UI 中必须显示 "DESTRUCTIVE" 标签，使用最强烈的视觉警告。风险标签必须出现在审批对话框的顶部，使用大号字体和醒目图标，而非隐藏在折叠区域或工具提示中。16 个 HIGH_RISK_COMMAND_PATTERNS 匹配的操作必须使用最强的视觉警告级别，包括红色边框面板、脉动动画和 "DESTRUCTIVE" / "HIGH RISK" 文字标签。
⛔ CRITICAL - 系统级破坏性操作 (rm -rf /, mkfs, dd)
⚠️ HIGH - 实例停止/隔离/删除 (stop_openclaw, quarantine_skill)
🔴 MEDIUM - 配置变更/技能操作 (apply_config_revision, install_skill)
🟢 LOW - 只读/信息收集 (health_check, collect_system_info)
3
Impact Scope Display - 影响范围展示
精确展示受影响资源，基于 Instance 模型和 NetworkPolicy
MEDIUM
审批界面必须展示操作的精确影响范围。基于 ClawManager Instance 模型，停止操作需展示: Name（实例名称）、Type（实例类型）、Status（当前状态）、CPUCores / MemoryGB（资源规格）、PodName / PodNamespace（K8s 位置）。对于涉及网络策略变更的操作，需展示 ClawManager NetworkPolicy 的 egress 限制作为影响范围: DNS (UDP/TCP:53 到 kube-system)、管理通道 (TCP:80,443,8443,9001,3128 到 system namespace)。隔离技能操作需列出受影响的技能列表和依赖该技能的活跃会话数。模糊描述如"将影响部分服务"是禁止的; 必须列出具体的实例名、技能名、会话数。

Source Attribution - 来源归因
展示操作发起者身份，基于 IssuedBy 字段
MEDIUM
审批界面必须清晰展示操作的发起者。基于 ClawManager InstanceCommand 的 IssuedBy 字段追踪审批人，AgentID 字段标识触发命令的 Agent。用户需要知道: 1) 谁发起了这个操作（人工还是 AI Agent）; 2) 操作是主动执行还是自动触发; 3) 如果是 AI Agent 发起，需展示其推理链摘要。来源归因帮助用户判断操作的可信度，防止恶意 Agent 通过模糊的来源描述获取审批。OpenClaw Agent 发起的操作必须明确标注 "AI 自动发起"，而非伪装为人工操作。

Side-Effect Disclosure - 副作用披露
强制展示潜在后果，包括级联影响
CRITICAL
审批界面必须包含"副作用"区域，列出操作的潜在后果。停止 OpenClaw 不仅影响目标实例，还会导致: 1) 所有活跃聊天会话断开; 2) 正在执行的推理任务终止; 3) 依赖该实例的技能变为不可用; 4) 配置快照可能过期。隔离技能的副作用包括: 1) 所有使用该技能的会话失去功能; 2) 依赖该技能的其他技能可能级联失效; 3) 技能隔离后的恢复需要重新安装和配置。ClawAegis Dispatch Guard 标记为 destructive-intent 的操作必须在副作用区域显示 "DESTRUCTIVE" 标签，并用红色高亮其不可逆性。副作用区域不可折叠、不可隐藏，必须作为审批界面的固定组成部分。
⚠️ 副作用警告 - quarantine_skill
● 技能 web-search-v3 将被隔离，所有会话立即失去搜索功能
● 依赖技能 research-assistant、fact-checker 将级联失效
● 3 个活跃会话 (session-28a, session-31c, session-45f) 受影响
● 隔离后恢复需要重新安装 + 重新配置授权，无法一键撤销

Confirmation Friction - 确认摩擦
高风险操作需输入确认文本，增加决策摩擦
HIGH
高风险操作不能仅靠点击按钮确认，必须要求用户输入特定确认文本。这创造必要的"决策暂停"时刻，防止用户在未认真思考的情况下快速点击通过。确认文本根据操作类型不同: stop_openclaw 需输入 "STOP"，quarantine_skill 需输入 "QUARANTINE"，apply_config_revision 需输入 "APPLY"。ClawAegis 的 16 个 HIGH_RISK_COMMAND_PATTERNS 匹配的操作需输入 "DESTROY"。确认输入框必须区分大小写，输入正确文本前确认按钮保持禁用状态。中等风险操作可使用双击确认代替文本输入，但仍然需要显式的两次点击，不能合并为一次。

7
No Dark Patterns - 禁止暗黑模式
无预勾选框、无紧迫感施压、无隐藏条款
CRITICAL
审批界面严禁使用任何暗黑模式 (Dark Pattern) 设计技巧。具体禁止: 1) 预勾选框 - 不得有任何默认选中的确认选项; 2) 紧迫感施压 - 不得使用 "限时操作"、"即将过期"、"立即处理" 等催促性文案; 3) 隐藏条款 - 所有可能的影响必须直接展示，不得放在折叠区、工具提示或链接中; 4) 快速审批按钮 - 不得提供 "一键审批"、"快速通过" 等跳过审查流程的按钮; 5) 视觉干扰 - 取消按钮不得使用弱化的视觉样式，必须与确认按钮同等可见; 6) 误导性默认 - 不得将高风险选项设为默认值; 7) 确认疲劳 - 同一操作不得在短时间内多次弹出确认框制造疲劳感，但核心确认步骤不可省略。
