持续合规审计与偏差修复系统

FR-68 持续检查节点、实例、权限、策略、配置和运行状态是否偏离安全基线，并提供修复建议和自动修正机制。系统通过 KSecure 的 38 个基线 Shell 脚本进行合规扫描（G_MODE=c），ClawManager 的 InstanceDesiredState 与 InstanceRuntimeStatus 对齐验证期望态一致性，KubeArmor 的 DefaultPosture 确保基础设施层面的默认安全策略合规。当检测到偏差时，KSecure 提供三种修复模式：自动加固（G_MODE=s）、人工审核和自动回滚（G_MODE=b），配合 ClawManager 的安全扫描实现闭环修复。

Continuous Compliance Pipeline
持续合规流水线

持续合规审计通过五阶段闭环流水线实现：扫描阶段使用 KSecure G_MODE=c 执行 38 个基线脚本检查当前配置状态；检测阶段将当前值与基线值比对识别偏差；分类阶段按风险等级（Critical/High/Medium/Low）对偏差进行分级；修复阶段根据修复策略选择自动加固（G_MODE=s）、人工审核或自动回滚（G_MODE=b）；验证阶段重新扫描确认偏差已被修复。ClawManager 的 InstanceConfigRevision 通过 SHA256 checksum 比对当前配置与期望配置，KubeArmor 的 DefaultPosture 确保文件/网络/能力默认策略符合安全要求。

KSecure Baseline Compliance
KSecure 基线合规检查

KSecure 包含 38 个 Shell 脚本，覆盖 5 大安全基线类别，支持三种运行模式（G_MODE）：检查模式（c）仅检测并报告偏差；加固模式（s）自动修复配置使其符合基线；回滚模式（b）将配置恢复到加固前的状态。每个脚本都实现了 PassMinLengthCheck/Reinforce/RollBack 三种函数，通过 gRPC 端点 baselineScan/baselineRepair/baselineBack 提供远程调用能力。

// KSecure G_MODE 三模式操作示例 function PassMinLenth() { [ "${G_MODE}" == 'c' ] && PassMinLengthCheckWithEcho [ "${G_MODE}" == 's' ] && PassMinLengthReinforce [ "${G_MODE}" == 'b' ] && PassMinLengthRollBack }

ClawManager 期望态对齐

ClawManager 通过 InstanceDesiredState 与 InstanceRuntimeStatus 的持续比对来检测运行时偏差。InstanceConfigRevision 为每个实例配置生成 SHA256 checksum，当当前配置的 checksum 与期望配置的 checksum 不一致时，标记为配置漂移。安全扫描模块定期评估实例的 RBAC 权限、网络策略和资源配额是否符合安全基线。偏差检测后触发告警，并提供修复建议或自动对齐到期望态。

KubeArmor Infrastructure Compliance
KubeArmor 基础设施合规

KubeArmor 通过 DefaultPosture 和安全策略确保基础设施层面的合规性。DefaultPosture 定义了文件、网络和能力三个维度的默认处置策略（audit 或 block），任何未明确允许的操作都将按默认策略处理。CIS/STIG 合规策略覆盖了关键系统目录（/bin/、/boot/、/sbin/、/usr/bin/）的文件完整性监控（FIM），确保系统文件不被未授权修改。DefaultPosture 的合规检查确保所有命名空间的默认策略一致。

KSecure 基线 gRPC 端点

Deviation Remediation Modes
偏差修复模式

KSecure 提供三种修复模式应对不同类型的配置偏差：自动加固（G_MODE=s）适用于安全的配置变更，人工审核适用于有风险的变更，自动回滚（G_MODE=b）在加固导致服务异常时恢复原状。修复决策基于偏差的风险等级和修复的安全性评估。

Safe 自动加固 — G_MODE=s (Auto-Reinforce)

对于安全的配置变更（如设置密码复杂度参数、限制文件权限、配置SSH安全参数等），系统自动执行加固操作使其符合基线。KSecure 的 baselineRepair gRPC 端点触发 G_MODE=s 模式，每个基线脚本的 Reinforce 函数自动修改配置。加固前自动备份当前配置以支持回滚。适用于 Low 和 Medium 风险等级的偏差。

// KSecure 自动加固示例 function PassMinLengthReinforce() { # 备份当前配置 cp /etc/pam.d/system-auth /etc/pam.d/system-auth.bak # 设置密码最小长度 sed -i "s/minlen=.*/minlen=$I_MIN_LENGTH/" /etc/pam.d/system-auth sed -i "s/^minlen.*/minlen = $I_MIN_LENGTH/" /etc/security/pwquality.conf # 验证加固结果 PassMinLengthCheckWithEcho }

Risky 人工审核 — Manual Review

对于可能影响业务连续性的配置变更（如修改密码策略导致用户被锁定、更改SSH配置导致连接断开等），系统不自动修复而是生成修复建议并提交人工审核。审核通过后可通过 baselineRepair 端点手动触发加固，或在批准后进入自动加固流程。适用于 High 和 Critical 风险等级的偏差。

Recovery 自动回滚 — G_MODE=b (Auto-Rollback)

当自动加固导致服务异常或配置不兼容时，系统自动执行回滚操作恢复到加固前的配置状态。KSecure 的 baselineBack gRPC 端点触发 G_MODE=b 模式，每个基线脚本的 RollBack 函数从备份中恢复原始配置。回滚操作会触发安全告警，确保运维团队知晓。回滚完成后自动执行一次基线扫描验证配置已恢复。

// KSecure 回滚示例 function PassMinLengthRollBack() { # 从备份恢复 if [ -f /etc/pam.d/system-auth.bak ]; then cp /etc/pam.d/system-auth.bak /etc/pam.d/system-auth rm /etc/pam.d/system-auth.bak fi # 验证回滚结果 PassMinLengthCheckWithEcho }
