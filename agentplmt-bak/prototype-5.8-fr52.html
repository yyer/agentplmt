Multi-Source Content Review Architecture
多源内容审核流程架构
OpenClaw + ClawAegis + ClawManager

对模型输出、工具返回和待执行内容进行四维审核——事实一致性、合规性、有害性和来源一致性，减少"看起来可信"的错误输出。审核层集成 ClawAegis Prompt Guard（PG-04 有害内容检测 + 14 种混淆模式 + 编码守卫）和 ClawManager RiskRule（正则匹配 + 风险分级 + 动作策略），实现从内容输入到最终决策的全链路安全审核。

Pipeline Stage Details
审核管道各阶段详解
① 事实一致性检查 (Factual Consistency)
与已知事实库交叉引用，检测模型输出中的事实性错误、矛盾陈述和虚构内容
工具返回结果与预期格式/范围的一致性校验（数据类型、值域、枚举范围）
待执行内容与当前任务目标的逻辑一致性校验（动作是否符合用户意图）
利用 ClawManager ModelInvocation 审计记录进行回溯验证，检测上下文中的事实漂移
② 合规性审核 (Compliance Review)
匹配 ClawManager RiskRule 正则模式，识别合规违规（PII 泄露、行业违规、策略冲突）
检查内容是否符合当前安全策略和监管要求（等保/数据安全法/行业规范）
AI Gateway 对扁平化消息进行敏感内容检测，标记合规风险点
RiskRule Action 映射：allow（低风险）→ block（高风险）→ require_approval（需人工）
③ 有害内容检测 (Harmful Content Detection)
PG-04 有害内容检测：暴力、歧视、欺诈、违法、仇恨等显式有害内容
14 种混淆模式识别：同音字替换、字符插入、空格填充、Unicode 等价、方向操控等
编码守卫：检测 base64/base32/hex/URL 编码的隐藏有害载荷
输出脱敏：识别并标记模型响应中的 API Key、Token、Credential 等敏感信息
④ 来源一致性验证 (Source Attribution)
验证内容来源的可追溯性：模型输出是否引用可信来源，工具返回是否来自授权端点
Exfiltration Guard sourceSignals 追踪：内容是否从受保护来源流出
transformSignals 检测：内容是否经过未授权的转换/加工
sinkSignals 监控：内容是否向未授权目标传递，检测未归属的声明和断言

Review Dimensions
四大审核维度规则
4 维度 × 多检测方法
📖
事实一致性 (Factual Consistency)
对模型输出、工具返回内容与已知事实库进行交叉引用，检测事实性错误、矛盾陈述和虚构内容。验证待执行内容与当前任务目标的逻辑一致性，利用 ModelInvocation 审计记录进行回溯验证。
检测方法：
知识库交叉引用 矛盾检测 幻觉识别 回溯验证 逻辑一致性
阈值：置信度 < 0.7 触发标记，< 0.4 阻断
动作：≥0.7 通过 0.4-0.7 标记 <0.4 阻断
⚖️
合规性审核 (Compliance Review)
匹配 ClawManager RiskRule 正则模式，识别 PII 泄露、行业违规和策略冲突。AI Gateway 对扁平化消息进行敏感内容检测，标记合规风险点。符合等保、数据安全法和行业规范要求。
检测方法：
RiskRule 匹配 PII 检测 监管合规校验 AI Gateway 扫描 策略冲突检测
阈值：RiskRule severity=low 放行，medium 标记，high/critical 阻断
动作：low 放行 medium 标记 high+ 阻断
🚨
有害内容检测 (Harmful Content)
PG-04 检测显式有害内容（暴力/歧视/欺诈/违法/仇恨），14 种混淆模式识别隐藏有害内容（同音字/字符插入/空格填充/Unicode等价/方向操控等），编码守卫检测 base64/base32/hex/URL 编码的隐藏有害载荷，输出脱敏标记 API Key/Token/Credential。
检测方法：
PG-04 检测 14 混淆模式 编码守卫 输出脱敏 语义分析
阈值：PG-04 score > 0.3 标记，> 0.7 阻断；编码异常直接阻断
动作：≤0.3 通过 0.3-0.7 标记 >0.7/编码 阻断
🔗
来源一致性验证 (Source Attribution)
验证内容来源的可追溯性，确保模型输出引用可信来源，工具返回来自授权端点。利用 Exfiltration Guard 的 sourceSignals/transformSignals/sinkSignals 三维信号追踪，检测未授权数据流动和未归属的声明断言。
检测方法：
来源追溯 sourceSignals transformSignals sinkSignals 未归属声明检测
阈值：来源可追溯且信号清零通过，信号异常标记，确认外泄阻断
动作：信号清零 通过 信号异常 标记 确认外泄 阻断

检测违反数据安全法的跨境数据传输建议，拦截未经合规评估的跨境数据操作
/transfer.*overseas/i /export.*data.*foreign/i Cross-border Data
