KubeArmor — IMA Configuration
KubeArmor IMA 配置
kubearmor
IMA 度量开关 enableIMA
启用 IMA 完整性度量，依赖 BPF LSM 内核支持。启用后 KubeArmor 将通过 eBPF 钩子收集文件哈希
依赖: BPF LSM 支持
内核须启用 CONFIG_BPF_LSM=y，可通过 cat /sys/kernel/security/lsm 确认 bpf 在列表中
LSM 钩子
bpf_file_ima_hash 挂钩点: ImaBprmCheckSecurity
哈希字段
ProcessHash ParentHash ResourceHash HashAlgo
Visibility 配置
IMA bool — eBPF per-namespace map 中 index 5 的可见性标志
当日志数据的 ctx.Hash == 1 时，IMA 哈希信息将包含在事件日志中
ClawManager — Config Revision Checksum
ClawManager 配置版本 SHA256 校验
clawmanager
Checksum 格式
sha256:a1b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6e7f8a9b0c1d2e3f4a5b6c7d8e9f0a1b2
格式为 sha256: 前缀 + 64位十六进制字符
数据模型: InstanceConfigRevision
字段类型说明
iduint主键 ID
instance_iduint关联实例 ID
revision_noint版本号（递增）
content_jsonjson配置内容 JSON
checksumstringSHA-256 校验和
statusstring版本状态（draft / published / activated）
published_bystring发布者
published_atdatetime发布时间
activated_atdatetime激活时间
计算方式
func computeChecksum(contentJSON string) string { hash := sha256.Sum256([]byte(contentJSON)) return "sha256:" + hex.EncodeToString(hash[:]) }
对 content_json 原始字节取 SHA-256，生成 64 字符十六进制校验和
ClawAegis — Self-Integrity Verification
ClawAegis 自身完整性校验
claw-aegis
启动时计算 12 个文件的 SHA-256 指纹
每次启动时计算以下文件的 SHA-256 哈希，与基线比对以检测篡改
index.ts runtime-api.ts openclaw.plugin.json package.json src/config.ts src/types.ts src/state.ts src/rules.ts src/scan-service.ts src/scan-worker.ts src/scan-worker.js src/handlers.ts
持久化存储
self-integrity.json
仅保存每个文件哈希的前 16 个 hex 字符，用于快速比对
存储格式示例
{ "index.ts": "a1b2c3d4e5f6a7b8", "runtime-api.ts": "b2c3d4e5f6a7b8c9", "openclaw.plugin.json": "c3d4e5f6a7b8c9d0", // ... 共 12 个文件，每个保存前 16 hex 字符 "src/handlers.ts": "d4e5f6a7b8c9d0e1" }

IMA Visibility Configuration
IMA 可见性配置
kubearmor
Per-namespace IMA visibility (eBPF map index 5)
Visibility Struct
type Visibility struct { // index 0-4: ProcessVisibility, FileVisibility, NetworkVisibility, ... IMA bool // index 5 }
当 IMA 字段为 true 时，eBPF 程序将在事件日志中附带文件哈希度量值
日志触发条件
当 ctx.Hash == 1 时，事件日志数据将包含 IMA 哈希字段： ProcessHash, ParentHash, ResourceHash, HashAlgo
IMA Log Format
IMA 日志格式示例
kubearmor
{ "processHash": "sha256:abc123...", "parentHash": "sha256:def456...", "resourceHash": "sha256:789abc...", "hashAlgo": "sha256" }
各字段说明：
processHash — 当前执行进程的 SHA-256 哈希（由 bpf_file_ima_hash 钩子计算）
parentHash — 父进程的 SHA-256 哈希，用于追踪进程链
resourceHash — 被访问文件/资源的 SHA-256 哈希
hashAlgo — 哈希算法标识，当前固定为 sha256
