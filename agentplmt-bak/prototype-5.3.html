FR-15 · P0
数据加密存储
加密存储架构
系统采用多层加密策略：
1. K8s Secret — API Key、凭据等敏感配置存储在 K8s Secret 中，由集群 etcd 加密。
2. 凭据代理 — AI Gateway 通过 SecretRefService 在代理时解析 K8s Secret，应用层不持久化明文。
3. .env 加密 — SecureClaw 凭据加固模块将 .env 加密为 .env.enc（AES-256-GCM）。
4. 字段级保护 — 数据库模型中 json:"-" 标签阻止敏感字段 API 暴露。

会话隔离架构
多层隔离确保 Agent 会话数据不互相泄露：
1. K8s Namespace 隔离 — 每个 Agent 运行在独立用户命名空间 clawreef-user-{userID}，Secret、Pod 等资源隔离
2. 会话令牌隔离 — agt_sess_ 会话令牌绑定实例，24h 滚动续期
3. Exfiltration Guard — 检测跨会话数据外泄链路（source → transform → sink）
4. KubeArmor 网络策略 — nftables 默认拒绝，仅白名单放行
