ClawManager 数据模型
Instance 配额字段映射
ClawManager API

以下字段来自 ClawManager Instance 模型，定义每个 Agent 实例的计算资源配额与硬件约束。KSecure/KubeArmor 基于 OSM（容器资源限制）强制执行 CPU/内存上限。

CPUCoresfloat64 — 实例分配的 CPU 核心数，支持小数（如 0.5 = 500m）。当前范围: 1.0 ~ 8.0 cores
MemoryGBint — 实例分配的内存容量（GB）。当前范围: 4 ~ 32 GB。KubeArmor relay 最低保障: 64Mi
DiskGBint — 实例分配的磁盘容量（GB）。当前范围: 50 ~ 500 GB
GPUEnabledbool — 是否启用 GPU 加速。启用时需配置 GPUType 和 GPUCount
GPUTypestring — GPU 型号标识，如 "NVIDIA A100"、"NVIDIA H100"。仅 GPUEnabled=true 时有效
GPUCountint — GPU 设备数量。当前范围: 0 ~ 4。仅 GPUEnabled=true 时有效
KubeArmor Helm 资源基线: relay requests cpu:10m memory:64Mi | controller requests cpu:10m memory:64Mi — 所有 Instance 配额不得低于此基线值。KSecure eBPF map 约束: ransomware_decoy max_entries=1024, app_path max_entries=10, RingBuffer 256KB。

ClawAegis 防护集成
ClawAegis 循环防护与资源约束
ClawAegis

ClawAegis 为多 Agent 协同过程提供循环防护、内存写入限制和技能扫描约束，防止 Agent 行为失控导致资源过载和级联失稳。以下参数与并发控制联动，超限时自动触发限流或阻断。

↻ Loop Guard 循环防护

检测 Agent 在短时间内重复执行相同操作的行为。对 write/edit/apply_patch/exec/bash/memory_store 操作类型，5 分钟内重复 3 次即触发循环告警与阻断。

write 循环检测
5 分钟内同一 Agent 对同一目标重复 write 操作达 3 次时触发限流
3 次 / 5min
edit 循环检测
5 分钟内同一 Agent 对同一文件重复 edit 操作达 3 次时触发限流
3 次 / 5min
apply_patch 循环检测
5 分钟内同一 Agent 对同一文件重复 apply_patch 操作达 3 次时触发限流
3 次 / 5min
exec 循环检测
5 分钟内同一 Agent 重复执行相同命令达 3 次时触发限流
3 次 / 5min
bash 循环检测
5 分钟内同一 Agent 重复执行相同 bash 命令达 3 次时触发限流
3 次 / 5min
memory_store 循环检测
5 分钟内同一 Agent 重复写入相同 memory key 达 3 次时触发限流
3 次 / 5min
M Memory 写入限制

限制 Agent 通过 memory_store 写入的数据量，防止单个 Agent 占用过多共享内存资源，导致其他 Agent 无法写入或读取性能下降。

MEMORY_WRITE_MAX_CHARS chars
MEMORY_WRITE_MAX_LINES lines
超过 8192 字符或 200 行的 memory_store 请求将被自动截断或拒绝，并记录到限流日志。
S 技能扫描限制 Skill Scan

限制 Agent 技能扫描的并发度、超时和冷却时间，防止扫描操作占用过多系统资源。文件大小限制防止单次扫描处理过大文件。

SKILL_SCAN_QUEUE_MAX tasks
SKILL_SCAN_TIMEOUT_MS ms
SKILL_SCAN_COOLDOWN_MS ms (5min)
SKILL_SCAN_FILE_MAX_BYTES bytes (100KB)

承载系统集成说明
ClawManager AI Gateway Token 限流 + CostRecord 成本核算。Instance 模型定义 CPU/Memory/Disk/GPU 配额，LLMModel 管理模型定价与安全路由，RiskRule 联动限流策略。
KSecure/KubeArmor KOS 容器资源限制强制执行。KubeArmor Helm 基线: relay requests cpu:10m memory:64Mi, controller requests cpu:10m memory:64Mi。KSecure eBPF 约束: ransomware_decoy max_entries=1024, app_path max_entries=10, RingBuffer 256KB。
ClawAegis Loop Guard (3 repeats/5min) 防循环、Memory 限制 (8192 chars/200 lines) 防溢出、Scan 限制 (16 queue/3s timeout/5min cooldown/100KB max) 防扫描风暴。
OpenClaw 跨实例任务调度与并发管理，配合 ClawManager 配额策略实现多 Agent 协同资源管控。
