KubeArmor PKI 认证流
CA → Server Cert → Client Cert → 双向验证 (mTLS)
KubeArmor

KubeArmor 通过 PKI 体系实现 mTLS 传输层双向认证。TLS 模式使用 RequireAndVerifyClientCert，强制客户端也必须出示有效证书。证书体系包含 CA 证书（RSA 4096, 3年有效期）、服务器证书（kubearmor.{ns}.svc.cluster.local）和客户端证书三层结构，通过 3 个 K8s Secret 管理。

🔐 Root CA — kubearmor-ca
算法: RSA 4096 | 有效期: 3 年 | CN: kubearmor-ca
K8s Secret: kubearmor-ca — 包含 ca.crt + ca.key
用于签发服务器证书和客户端证书
🖥️ Server Cert — kubearmor-relay-server
DNS SANs: kubearmor.{ns}.svc.cluster.local, kubearmor, kubearmor.{ns}, localhost
K8s Secret: kubearmor-relay-server — 包含 server.crt + server.key
TLS 配置: RequireAndVerifyClientCert — 强制客户端证书验证
🔑 Client Cert — kubearmor-client
用途: Agent 连接 KubeArmor Relay Server 时出示
K8s Secret: kubearmor-client — 包含 client.crt + client.key
