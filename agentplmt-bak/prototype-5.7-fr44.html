Agent 注册流程 (5 步)
POST /api/v1/agent/register
1
Bootstrap Token 生成
当 Instance 创建时，ClawManager 自动为每个 openclaw 类型实例生成唯一的 Bootstrap Token（前缀 agt_boot_），写入 instances.agent_bootstrap_token 字段。Token 使用 crypto/rand 生成 24 字节随机数经 hex 编码，确保不可预测。
// internal/services/instance_agent_service.go func generatePrefixedToken(prefix string) (string, error) { bytes := make([]byte, 24) if _, err := rand.Read(bytes); err != nil { return "", err } return prefix + "_" + hex.EncodeToString(bytes), nil } // 生成结果示例: agt_boot_a1b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6e7f8a9b0c1d2e3f4
2
Token 注入 (环境变量)
Kubernetes Pod 创建时通过 KubeArmor Mutation Webhook 注入安全注解，ClawManager 将 Bootstrap Token 和连接信息以环境变量方式注入 Agent Pod。Agent 启动后读取以下环境变量完成初始化配置：
CLAWMANAGER_AGENT_ENABLEDtrue
CLAWMANAGER_AGENT_BASE_URLhttps://clawmanager.example.com/api/v1
CLAWMANAGER_AGENT_BOOTSTRAP_TOKENagt_boot_a1b2c3d4e5f6...d2e3f4
CLAWMANAGER_AGENT_INSTANCE_ID42
CLAWMANAGER_AGENT_PROTOCOL_VERSIONv1
KubeArmor Mutation Webhook: kubearmor-controller-mutating-webhook-configuration 监听 Pod CREATE/UPDATE 事件，通过 /mutate-pods 端点自动注入安全注解和 AppArmor Profile，确保 Agent Pod 在可信计算基线内启动。
3
Agent 注册 (POST /register)
Agent 使用 Bootstrap Token 作为 Bearer 凭证向 ClawManager 发起注册请求。服务端依次执行：Token 验证 → Instance 查找 → Protocol Version 校验 → Instance Type 检查 → Agent 记录创建/更新。任何步骤失败均返回 401/400/403 错误。
REQUEST
POST /api/v1/agent/register Authorization: Bearer agt_boot_a1b2... { "instance_id": 42, "agent_id": "openclaw-agent-7f8a9b", "agent_version": "1.2.0", "protocol_version": "v1", "capabilities": [ "command_execution", "file_operations", "skill_management", "state_reporting", "heartbeat" ], "host_info": { "hostname": "openclaw-pod-42", "os": "linux", "arch": "amd64", "pid": 1, "node": "worker-node-03" } }
RESPONSE 200
{ "session_token": "agt_sess_e3f4a5b6c7d8...", "session_expires_at": "2026-05-02T08:30:00Z", "heartbeat_interval_seconds": 15, "command_poll_interval_seconds": 5, "server_time": "2026-05-01T08:30:00Z" }
4
Session Token 签发
注册成功后，服务端生成 Session Token（前缀 agt_sess_），有效期 24 小时，写入 instance_agents.session_token。Session Token 用于后续所有 API 调用的身份认证，每次心跳成功后自动续期（滚动窗口 24h），防止长期运行 Agent 意外过期锁死。数据库 UNIQUE KEY 约束 uk_instance_agents_session_token 确保每个 Session Token 全局唯一。
5
心跳启动
Agent 以 15 秒间隔发送心跳，服务端根据心跳更新 last_heartbeat_at 并推导状态：≤45s = online，45s~120s = stale，>120s = offline。心跳同时上报 OpenClaw 运行状态、当前配置版本和运行摘要。

Bootstrap Token Validation
Token-Instance 绑定
Bootstrap Token 与 Instance 一对一绑定，注册时必须同时满足：1) Token 可查到对应 Instance；2) 查到的 Instance.ID 与请求中 instance_id 精确匹配。两者任一不满足即返回 "invalid agent bootstrap token"，防止 Token 泄露后被跨实例盗用。
GetByAgentBootstrapToken() instance.ID == req.InstanceID crypto/rand 24-byte
ProtocolVersion == "v1"
协议版本固定
仅允许协议版本 v1 的 Agent 注册。所有其他版本（包括旧版 v0、伪造 v2、空字符串）均被拒绝并返回 "unsupported agent protocol version"。这防止攻击者通过协议降级绕过新版安全校验逻辑。
AgentProtocolVersionV1 = "v1" ProtocolVersion != "v1" → REJECT
Instance.Type == "openclaw"
类型限制
仅允许 openclaw 类型的 Instance 注册 Agent，其他类型（如标准虚拟桌面、测试实例等）调用注册接口将收到 "agent registration is only supported for openclaw instances" 错误。这限制了 Agent 注册的攻击面，仅对需要 Agent 管理的智能体实例开放。

UNIQUE KEY 约束 uk_instance_agents_instance 确保同一 Instance 不能注册多个 Agent。若检测到试图使用不同 AgentID 注册同一 Instance 的行为（即已有 Agent 在线时新的注册请求携带不同 AgentID），触发 Agent 替换告警并记录新旧 AgentID 对比，需要人工确认是否为合法替换场景。
