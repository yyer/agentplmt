Role Hierarchy
管理面角色层级
当前系统仅区分 admin / user 两级（见 AdminAuth 中间件 user.Role != "admin" 检查）。FR-75 增强为五级角色体系，逐级权限递减，上级自动包含下级权限。
★
Super Admin 超级管理员
系统最高权限，可管理所有资源和用户。唯一可执行用户角色变更和紧急访问操作的角色。
Instance CRUD Command Issue Config Revision Risk Rule Mgmt User Mgmt Audit Access Emergency Access
🛡
Security Admin 安全管理员
安全策略和风险管理专属角色。可管理 RiskRule、安全扫描配置、KubeArmor 策略，不可管理用户。
Instance Read Risk Rule Mgmt Security Config KubeArmor Policy Audit Access User Mgmt Instance Delete
⚙
Operator 运维操作员
日常运维操作角色。可启停实例、下发命令、发布配置修订，不可修改安全策略或管理用户。
Instance Start/Stop Command Issue Config Revision Publish Instance Read Risk Rule Mgmt User Mgmt Stop Production
📄
Auditor 审计员
只读审计角色。可查看所有管理操作日志、审批记录、合规报告，不可修改任何资源。
Instance Read Audit Read Risk Rule Read Security Config Read Any Write Command Issue
👁
Viewer 查看者
最基础的只读角色。仅可查看仪表盘和自身实例状态，无权查看审计日志或安全配置。
Dashboard Own Instance Read All Instance Read Audit Read Any Write

KSecure Privilege Enforcement
KSecure 根权限强制与多人认证
ksecure
KSecure 守护进程启动时强制校验 os.Geteuid() != 0，非 root 权限直接退出。管理面操作结合 KSecure 的 root 权限要求和多人认证机制，确保高危管理操作的安全性。
KSecure Root Check
if os.Geteuid() != 0 {
  kg.Printf("Need to have root privileges to run %s\n", os.Args[0])
  return
}
管理面操作需 root 权限绑定
高危管理操作（KubeArmorConfig 变更、Agent 证书管理）需通过 KSecure 以 root 权限执行
多人认证 (M-of-N)
关键管理操作需至少 M 人（M >= 2）独立认证后方可执行，防止单点权限滥用

Dual Approval
高危操作双人审批规则
P0
删除实例
CRITICAL 2x Admin
操作: DELETE /api/v1/instances/:id
审批要求: 需 2 名 Admin 级别以上用户独立确认。其中至少 1 名为 Super Admin 或 Security Admin。
超时: 30 分钟内未获双人审批则自动取消
回退策略: 超时后操作自动作废，通知发起者和安全团队。紧急情况可由 Super Admin 单人审批（需记录紧急原因）。
停止生产实例
HIGH SecAdmin + Operator
操作: POST /api/v1/instances/:id/stop (production 标签实例)
审批要求: Security Admin + Operator 联合确认。Security Admin 评估安全影响，Operator 确认运维必要性。
超时: 15 分钟
回退策略: 超时后操作取消。非生产实例的停止操作仅需 Operator 单人确认。
修改风险规则
CRITICAL 2x Security Admin
操作: PUT /api/v1/admin/risk-rules / DELETE /api/v1/admin/risk-rules/:ruleId
审批要求: 2 名 Security Admin 独立确认。风险规则变更直接影响全平台安全策略，需双重确认防止误操作或恶意篡改。
超时: 60 分钟
回退策略: 超时后操作取消，RiskRule 保持原状。规则变更前需展示 diff 对比。
修改 DefaultPosture
HIGH SecAdmin + Operator
操作: 修改 KubeArmorConfig CRD defaultPosture 字段 (file/network/capabilities action)
审批要求: Security Admin + Operator 联合确认。DefaultPosture 变更影响所有管理面 Pod 的默认行为，需安全与运维双方评估。
超时: 45 分钟
回退策略: 超时后操作取消，KubeArmorConfig 保持当前 defaultPosture。变更前需预览影响范围。
吊销 Agent 证书
HIGH Security Admin 审批
操作: POST /api/v1/agent/register (证书吊销/重新签发) 或删除 Agent 注册信息
审批要求: Security Admin 审批确认。Agent 证书吊销将导致对应实例失去与管理面的通信能力，需确认安全风险。
超时: 20 分钟
回退策略: 超时后操作取消。证书吊销前需确认关联实例已安全迁移或停用。
InstanceCommand & DesiredState Tracking

claw-manager
ClawManager 已通过 InstanceCommand.IssuedBy 和 InstanceDesiredState.UpdatedBy 字段跟踪操作发起人。双人审批在此基础上增加 ApprovedBy 字段记录第二审批人。

IssuedAt: time.Time   // 操作发起时间
CommandType: string  // 命令类型

+ ApprovedBy: *int    // 双人审批人 ID (FR-75)

紧急访问需 Super Admin
