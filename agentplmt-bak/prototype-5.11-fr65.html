TraceID Propagation
TraceID 传播与关联机制
🔗 主动传播链路
trace_id 由 AI Gateway 在请求入口生成
→ 传播至 ModelInvocation 记录（TraceID, SessionID, RequestID）
→ 传播至 CostRecord（TraceID, SessionID, RequestID 关联决策成本）
→ 传播至 InstanceCommand（IssuedBy 携带 trace_id）
→ 传播至 SecureClaw 审计事件（gateway.risk.detected）
🔄 被动关联链路
KubeArmor Alert/Log 通过 PodName + Timestamp 关联
KSecure SecLog 通过 HostName + Path 关联
关联策略：时间窗口 ≤ 5s，同一 Pod/Host 匹配
InstanceConfigRevision 通过 SHA256 checksum 校验完整性
关联缺失时标记为 Gap 待人工确认

Correlation Mode
链路关联模式
执行关联策略:
Enforce
Observe
Off

Enforce 模式下，链路关联缺失将阻断后续操作并触发告警；Observe 模式仅记录告警不阻断；Off 模式关闭关联检查。建议生产环境使用 Enforce 模式确保端到端可追溯性。

Enforce TraceID 传播规则
Enforce
Observe
Off

规则描述: trace_id 必须在所有链路节点间传播，ClawManager 内部组件通过显式 TraceID 字段传递，跨系统通过 PodName + Timestamp 或 HostName + Path 被动关联。

传播路径: AI Gateway (生成) → ModelInvocation (TraceID) → CostRecord (TraceID) → InstanceCommand (IssuedBy) → SecureClaw (trace_id)

被动关联: KubeArmor Alert/Log 通过 PodName + Timestamp(≤5s窗口) ↔ KSecure SecLog 通过 HostName + Path

违规动作: Enforce 阻断操作并告警 | Observe 仅记录 | Off 忽略

Enforce 跨系统关联规则
Enforce
Observe
Off

规则描述: ClawManager ↔ KubeArmor ↔ KSecure 之间的事件必须能通过关联键建立映射关系。每个系统的事件必须有唯一标识且包含可关联字段。

ClawManager ↔ KubeArmor: InstanceID/ContainerName ↔ PodName, Timestamp 窗口 ≤ 5s

ClawManager ↔ KSecure: InstanceID/HostName ↔ HostName, Path 匹配

KubeArmor ↔ KSecure: PodName/NamespaceName ↔ HostName, PID 匹配

关联超时: 若 30s 内未完成关联，标记为 Gap 并触发重新扫描

Observe 时序排序规则
Enforce
Observe
Off

规则描述: 追踪链路中的事件必须按时间顺序排列。跨系统事件因时钟偏移允许 ≤ 2s 的时间倒流，超过此阈值的事件标记为时序异常。

排序策略: 主排序 = Timestamp (毫秒精度), 次排序 = 系统优先级 (ClawManager > KubeArmor > KSecure > SecureClaw)

时钟偏移容忍: 同一 Host 内 ≤ 500ms, 跨 Host ≤ 2s, 跨 Cluster ≤ 5s

违规动作: Observe 标记时序异常事件但不阻断，等待人工确认

Enforce Gap 检测规则
Enforce
Observe
Off

规则描述: 追踪链路中缺失的步骤必须被检测和标记。Gap 检测基于预期链路模板: 用户请求 → AI Gateway → 模型决策 → 技能调用 → 工具执行 → 文件访问 → 网络访问 → 处置动作。

检测策略: Step N 完成后，若 ≤ 30s 内未检测到 Step N+1，标记为 Gap。对于可选步骤（文件访问/网络访问），允许跳过但需记录"无相关事件"。

Gap 级别: Critical — 必选步骤缺失（如 AI Gateway → 模型决策）| High — 关键步骤缺失（如技能调用 → 工具执行）| Medium — 可选步骤缺失

违规动作: Enforce Critical/High Gap 阻断操作 | Medium Gap 记录告警

Enforce 配置变更审计规则
Enforce
Observe
Off

规则描述: InstanceConfigRevision 每次变更必须生成新的 RevisionNo 并计算 SHA256 checksum。PublishedAt → ActivatedAt 的时间差用于审计配置激活延迟。

完整性校验: ContentJSON 的 SHA256 checksum 必须与存储值一致，任何不匹配视为篡改

违规动作: Enforce checksum 不匹配阻断激活并告警
