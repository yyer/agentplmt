安全事件联动处置与实例自愈恢复架构

FR-69 基于风险检测结果自动执行断网、隔离、冻结、摘流量、重建和恢复等动作，实现从威胁检测到自动处置再到审批恢复的全闭环响应。系统由 ClawManager（Agent 命令下发 — quarantine_skill / stop_openclaw / restart_openclaw + K8s 实例管理）+ KSecure（主机一键网络隔离 + ProcessBlackType 进程清理）+ KubeArmor（DefaultPosture audit→block 硬化 + NetworkPolicyEnforcer nftables）协同实现，确保高危事件在秒级触发自动隔离，恢复操作需经审批流程后方可执行。

ClawManager Commands
🌐
断网 Network Disconnect
Critical
执行方式: ClawManager 下发 K8s NetworkPolicy，全量拒绝出站流量
白名单: DNS(53) + 系统端口(80,443,8443,9001,3128) 可选保留或全拒
KSecure: 主机级网络隔离 iptables/nftables
KubeArmor: NetworkPolicyEnforcer (nftables) 强制执行
触发条件: 数据外泄链、C2通信检测
恢复: 需审批后恢复 NetworkPolicy egress 规则
🛑
隔离 Quarantine
High
执行方式: ClawManager 下发 quarantine_skill 命令
效果: Agent 被标记为隔离态，禁止执行任何 Skill
关联: handle_skill_risk + disable_skill 配合
触发条件: 高危风险检测、漂移评分 >0.8
恢复: 需审批解除隔离状态
❄
冻结 Freeze
High
执行方式: stop_openclaw + DesiredPowerState=stopped
效果: OpenClaw 实例停止运行，Pod 保留不删除
ClawManager: InstanceDesiredState 切换至 stopped
触发条件: 持续循环（Loop Guard 5+触发）、Rogue Agent
恢复: restart_openclaw + DesiredPowerState=running
🛢
摘流量 Drain Traffic
Medium
执行方式: 从 K8s Service 后端移除 Pod
效果: 实例不再接收新流量，已有连接优雅关闭
K8s: 调整 Service selector / Endpoint 切片
触发条件: 中等风险事件、可疑行为观察
恢复: 重新加入 Service 后端，流量逐步恢复
🔧
重建 Rebuild
Critical
执行方式: 删除并重新创建 Pod
效果: 实例从干净镜像重建，丢弃所有运行时状态
ClawManager: K8s Pod delete + recreate
触发条件: 勒索软件检测、实例被完全入侵
前置: 证据固化完成（FR-70）+ 审批通过
恢复: 新 Pod 就绪后自动加入服务
✅
恢复 Restore
Recovery
执行方式: restart_openclaw + 恢复 NetworkPolicy
效果: 实例恢复运行，网络策略恢复到正常状态
ClawManager: DesiredPowerState→running, egress restore
KSecure: 解除主机网络隔离
KubeArmor: DefaultPosture 恢复 audit 模式
审批: Critical/High 事件恢复必须审批
系统集成架构 — ClawManager + KSecure + KubeArmor + OpenClaw
联动处置的核心在于 ClawManager 作为编排中枢：检测事件从 ClawAegis（Loop Guard 持续触发 → 自动冻结）和 KubeArmor/KSecure（安全告警 → 自动隔离）汇聚，ClawManager 根据预设规则自动选择处置动作组合，通过 InstanceCommand 下发 quarantine_skill、stop_openclaw、restart_openclaw、handle_skill_risk、disable_skill 等命令，同时通过 K8s API 管理 NetworkPolicy 和 Pod 生命周期。KSecure 提供主机级网络隔离兜底，KubeArmor 通过 DefaultPosture 切换和 NetworkPolicyEnforcer 提供内核级策略执行。
🎯 ClawManager 命令体系
InstanceCommand 类型:
• quarantine_skill — Agent 隔离，禁止 Skill 执行
• stop_openclaw — 停止 OpenClaw 实例运行
• restart_openclaw — 重启 OpenClaw 实例
• handle_skill_risk — 风险 Skill 处置指令
• disable_skill — 禁用指定 Skill

InstanceDesiredState:
• DesiredPowerState: running / stopped
• 配合 InstanceCommand 实现冻结/恢复
🛡 K8s + KSecure + KubeArmor 协同
K8s NetworkPolicy:
• 断网: egress 全量 deny (可保留 DNS 53)
• 限制: 仅允许 80,443,8443,9001,3128
• 恢复: restore 原始 egress 规则

KSecure: 主机级 iptables/nftables 网络隔离
KubeArmor: DefaultPosture audit→block 即时硬化
• NetworkPolicyEnforcer (nftables) 执行策略
