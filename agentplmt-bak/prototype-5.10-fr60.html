Agent Behavior Baseline System
代理行为基线建立系统

FR-60 为不同 Agent 建立预期目标、工具使用模式、资源消耗模式和行为边界基线，用于识别长期偏航、异常自主性和行为漂移。基线系统通过持续采集 ClawManager 的 ModelInvocation、InstanceCommand、CostRecord 和 Agent 状态数据，自动生成行为画像并建立可量化的基线参考，当 Agent 行为偏离基线时触发告警。基线策略通过 Open Policy Agent (OPA) 执行，确保基线比较的客观性和可审计性。

Baseline Architecture Pipeline
基线建立流程架构

基线系统通过五阶段流水线从原始数据到告警输出：数据采集层汇聚 ClawManager 各数据源的 Agent 行为记录；画像生成层基于统计模型提取工具使用频率、资源消耗范围、命令类型分布和风险模式特征；基线比较层将实时行为与已建立的基线进行 OPA 策略评估；偏离检测层识别显著偏离基线的行为模式；告警层根据偏离严重程度和持续时间触发分级告警。

Acceptance Criteria
FR-60 验收标准
AC-60.1 Agent 行为画像可生成
系统可为每个注册 Agent 自动生成行为画像，画像包含工具使用模式、资源消耗模式、命令频率分布、风险模式特征和行为边界定义五类核心数据。画像生成基于至少 30 天学习期和 1000+ 数据点。
AC-60.2 画像包含工具使用和资源消耗模式
行为画像必须包含 InstanceCommand 历史分析的工具使用频率分布、InstanceCommand 生命周期的执行时长模式、ModelInvocation/CostRecord 的 token 消耗和成本趋势、AI Gateway 的风险检测匹配分布。
AC-60.3 偏离基线有告警
当 Agent 行为偏离已建立基线时，系统通过 OPA 策略评估触发分级告警。告警包含偏离类型、偏离幅度、基线参考值、当前值、置信度和建议操作。支持 enforce/observe/off 三种运行模式。

🚫 行为边界定义

行为边界基于 InstanceAgent 的 CapabilitiesJSON 和 OPA 策略定义，限定 Agent 的允许操作范围。超出 Capabilities 声明的行为、违反 OPA 策略的行为、或长期偏离基线的行为均视为边界越界，触发严重告警。

允许操作范围 (CapabilitiesJSON)
✓ start_openclaw ✓ stop_openclaw ✓ update_config ✓ health_check ✓ get_status ✗ delete_instance ✗ modify_permissions ✗ access_secrets
偏离容忍阈值
• 工具使用频率偏离: ≤ 15% (观察) / > 15% (告警)
• Token 消耗偏离: ≤ 20% (观察) / > 20% (告警)
• 成本偏离: ≤ 25% (观察) / > 25% (告警) / > 50% (严重)
• 风险等级偏离: 超出基线分布 2 个标准差 (告警)
• 能力越界: 任何超出 Capabilities 的行为 (严重)
OPA 策略：agent_baseline_compliance 策略包包含 12 条规则，涵盖工具使用、资源消耗、风险等级、行为边界四大维度。每条规则定义了基线参考值、偏离阈值和触发动作，通过 OPA Rego 引擎实时评估。

Baseline Deviation Detection Rules
基线偏离检测规则

基线偏离检测规则定义了四类偏离检测维度：工具使用偏离、资源消耗漂移、风险模式变化和行为边界越界。每条规则包含检测方法、阈值设定、基线参考和触发动作，通过 OPA Rego 策略引擎实时评估。规则支持三种运行模式：enforce（强制执行+阻断）、observe（观察记录）和 off（关闭）。

工具偏离 工具使用偏离检测 InstanceCommand 强制
检测方法
对比 InstanceCommand 历史中各命令类型的使用频率与基线频率，计算卡方检验或 KL 散度。检测异常命令类型（基线中未出现的命令）和异常频率突变（偏离基线 > 15%）。
偏离阈值
• 频率偏离 > 15%: 中度告警
• 频率偏离 > 30%: 高度告警
• 出现基线外命令: 高度告警
• 出现禁止命令: 严重告警

ClawAegis Integration
ClawAegis 守护集成配置

基线系统与 ClawAegis 的三个守护模块深度集成：Loop Guard、Dispatch Guard 和 Exfiltration Guard。每个守护模块的触发频率和检测模式纳入风险基线画像，偏离基线的触发模式变化自动联动风险模式变化检测规则。

Loop Guard
• ALLOW_COUNT: 3
• TTL_MS: 300000 (5 分钟)
• 跟踪键: sessionKey|runId|stableArgsKey
• 基线触发频率: ≤ 2 次/周
• 偏离检测: 超出基线触发 baseline_loop_guard_anomaly
Dispatch Guard
• 模式组: 4 组检测模式
• 检测维度: bypass-tool-call, destructive-intent, repeated-identical-calls, encoded-command
• 基线触发频率: ≤ 1 次/周
• 新模式组出现: 触发 baseline_dispatch_pattern_change
Exfiltration Guard
• 信号: sourceSignals, transformSignals, sinkSignals
• 链路模式: encoded-secret-outbound-chain, sensitive-source-transform-sink-chain
• 基线触发频率: ≤ 1 次/周
• 新外泄链路: 触发 baseline_exfil_chain_change
