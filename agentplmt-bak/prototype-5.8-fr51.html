Pre-Filter: Dispatch Guard
ClawAegis Dispatch Guard 破坏性命令拦截
前置拦截

Dispatch Guard 在命令到达审批流程之前进行预过滤，直接拦截已知的破坏性命令模式。这些命令不需要进入人工确认流程，因为它们被系统策略明确禁止。拦截结果可作为审计日志和风险分析的输入，帮助人工审批人做出更准确的判断。

拦截的破坏性命令组
✗
OpenClaw 实例操作
拦截 reset, uninstall, remove 等实例级破坏命令。这些操作不可逆，直接由 Dispatch Guard 阻断，不进入审批流程。
✗
Skills 危险操作
拦截 update, delete, remove, uninstall, sync 等技能级操作。技能变更影响全局行为，需严格管控。
✗
Plugins 危险操作
拦截 remove, uninstall, delete 等插件级操作。插件卸载影响系统功能完整性。
✗
系统级命令
拦截 shutdown, stop, kill, restart 等系统级命令。系统停机影响所有服务可用性。

Command Block Patterns
ClawAegis Command Block 16 种高风险命令模式
16 模式

Approval Chain Architecture
审批链配置与升级机制
👤
单人审批 (Single)
适用于高风险操作（如 stop_openclaw, disable_skill）。一位授权审批人确认即可放行。审批人需具有对应操作类型的审批权限，超时自动拒绝。
级别: Level 1
超时: 1800s (30min)
回退: reject
适用: stop, disable, quarantine
👤👤
双重审批 (Dual)
适用于关键操作（如 uninstall_skill, remove_skill, 对外付款）。需要两位不同授权人独立确认，防止单一审批人误操作。两位审批人不可为同一人。
级别: Level 1 + Level 2
超时: 3600s (1h)
回退: reject + escalate
适用: uninstall, remove, payment
🔗
审批链 (Chain)
适用于极关键操作（如安全策略放开、大规模数据导出）。L1 审批人确认后进入 L2，L2 确认后方可执行。任一级别拒绝即终止。支持超时自动升级到上一级。
级别: L1 → L2 → L3
超时: 7200s (2h)
回退: reject + alert
适用: policy change, bulk export
Self Protection Integration
ClawAegis Self Protection 与人工确认联动

ClawAegis Self Protection 维护受保护路径、受保护技能和受保护插件列表。任何对受保护资源的操作自动触发人工确认流程，且审批级别提升至 Dual 或 Chain。受保护资源操作不可由单人审批放行。

📁 protectedPaths
/etc/clawmanager /var/lib/openclaw /opt/aegis/config /data/secrets
删除/修改受保护路径需 Chain 审批
🧩 protectedSkills
security-scanner audit-logger auth-validator risk-analyzer
卸载/删除受保护技能需 Dual 审批
🔌 protectedPlugins
dispatch-guard loop-guard exfil-guard prompt-guard
移除/禁用受保护插件需 Chain 审批
Acceptance Criteria
验收标准
✓
AC-1: 高影响操作必须人工确认
删除、停服、批量变更、对外付款、导出敏感数据、策略放开等操作必须经过人工确认后方可执行。ClawManager InstanceCommand 高风险类型（stop_openclaw, quarantine_skill, handle_skill_risk, disable_skill, uninstall_skill, remove_skill）在 pending 阶段插入审批门控。
✓
AC-2: 确认记录可审计
每次人工确认操作生成完整审计记录：操作类型、目标实例、发起人（IssuedBy）、审批人、审批结果、时间戳、审批意见。支持通过 Request ID 追溯完整审批链路。
✓
AC-3: 支持审批链配置
支持单审批 (Single)、双审批 (Dual)、审批链 (Chain) 三种审批模式，可按操作类型和风险等级配置。审批链支持多级升级和超时自动回退。ClawAegis Self Protection 受保护资源操作自动提升审批级别。

高风险操作分类规则
CRITICAL 关键操作
删除、停服、OpenClaw reset/uninstall/remove、策略放开、敏感数据导出、对外付款。强制 Chain/Dual 审批，超时自动拒绝。
HIGH 高风险操作
disable_skill, quarantine_skill, handle_skill_risk, 批量配置变更, 插件卸载。需 Single/Dual 审批。
MEDIUM 中风险操作
install_skill, update_skill, restart_openclaw 等。默认记录日志，可选开启人工确认。
审批级别配置
Single 审批 - 超时 (秒)
单审批超时时间。默认 1800s (30min)。
Dual 审批 - 超时 (秒)
双审批超时时间。默认 3600s (1h)。
Chain 审批 - 超时 (秒)
审批链超时时间。默认 7200s (2h)。
金融操作 - 超时 (秒)
对外付款确认超时。建议不低于 3600s。

升级与回退规则
超时自动升级
L1 审批超时后自动升级到 L2，L2 超时后升级到 L3。升级时发送通知给下一级审批人。
超时自动拒绝
审批链最终超时后，自动拒绝操作请求。拒绝记录写入审计日志，触发告警通知。
超时回退动作
审批链最终超时后的处理动作。建议生产环境使用 Reject。
升级通知通道
邮件通知 Webhook Slack 钉钉 + 添加通道
Dispatch Guard 拦截模式配置
启用 Dispatch Guard 预过滤
在审批流程前拦截已知破坏性命令模式。拦截的操作不进入审批队列，直接阻断。
拦截但允许进入审批
Dispatch Guard 拦截后不直接阻断，而是标记为"需审批"并进入人工确认流程。适用于需要灵活管控的场景。
拦截组:
openclaw: reset, uninstall, remove
skills: update, delete, remove, uninstall, sync
plugins: remove, uninstall, delete
system: shutdown, stop, kill, restart
Command Block 高风险命令模式
启用 Command Block 拦截
16 种 HIGH_RISK_COMMAND_PATTERNS 拦截。与 Dispatch Guard 互补，覆盖文件系统破坏、权限提升、网络外泄等模式。
拦截后触发人工确认
Command Block 拦截的命令可选择进入人工确认流程（而非直接阻断），适用于需要灵活处理的场景。
当前模式数量: 16
可通过 API 或配置文件添加自定义 HIGH_RISK_COMMAND_PATTERNS。
Self Protection 受保护资源配置
protectedPaths (受保护路径)
/etc/clawmanager /var/lib/openclaw /opt/aegis/config /data/secrets + 添加路径
受保护路径的删除/修改操作自动提升至 Chain 审批。
protectedSkills (受保护技能)
security-scanner audit-logger auth-validator risk-analyzer + 添加技能
受保护技能的卸载/删除操作自动提升至 Dual 审批。
protectedPlugins (受保护插件)
dispatch-guard loop-guard exfil-guard prompt-guard + 添加插件
受保护插件的移除/禁用操作自动提升至 Chain 审批。
通知设置
新审批请求通知
当有新的高风险操作审批请求时，通知相关审批人。
审批截止前预警
在审批超时截止前 60s 发送预警通知。
审批结果通知
当审批决策完成后，通知操作发起人决策结果。
Dispatch Guard/Command Block 拦截通知
当破坏性命令被拦截时，通知安全管理员。
