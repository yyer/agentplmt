Anti-Replay Architecture
防重放架构总览
强制模式 ClawManager + OpenClaw

防重放机制确保每条协同消息携带唯一标识（nonce）、精确时间戳、有效会话标识和任务窗口，使旧消息无法被重新注入系统造成误执行、错误授权或跨上下文污染。ClawManager 通过 IdempotencyKey 机制保证命令幂等性，结合时间戳校验和会话绑定形成三层防线。

Layer 3
会话标识绑定
强制

消息必须携带有效会话标识（SessionToken），接收端验证会话是否存在且未过期。跨会话的消息被拒绝，防止上下文污染和会话劫持后的重放攻击。

  REJECT (会话过期)

newExpiry = now + 24h

ClawManager IdempotencyKey
命令幂等性机制详解
InstanceCommand

ClawManager 的 InstanceCommand 模型内置 IdempotencyKey 字段作为天然的重放防护机制。当客户端提交命令时，系统先按 (instance_id, idempotency_key) 查重，若已存在则直接返回已有命令而不重复创建，从根本上杜绝了命令重放的可能性。系统自动为各类操作生成语义化的 key：

自动生成规则
apply-config-revision-{revID} — 配置应用
install-skill-{instID}-{skillID} — 技能安装
remove-skill-{instID}-{skillID} — 技能移除
collect-skill-pkg-{instID}-{hash} — 技能包收集
{commandType}-{unixNano} — 通用回退
命令生命周期时间线
IssuedAt — 命令创建时间
DispatchedAt — 下发至 Agent 时间
StartedAt — Agent 开始执行时间
FinishedAt — 执行完成时间
TimeoutSeconds — 超时阈值 (默认 300s)
防重放保证：相同的 IdempotencyKey 只能创建一次命令。即使攻击者截获并重放完整的命令请求，服务端会识别重复 key 并返回首次创建的命令，不会产生第二次执行。配合 TimeoutSeconds 可确保过期命令不再被调度。
ClawAegis Loop Guard Integration
循环防护与重放检测联动
ClawAegis

ClawAegis 的 Loop Guard 模块提供循环行为检测能力，与 Anti-replay 机制形成互补。Loop Guard 检测同一会话中重复的 tool 调用模式，当检测到潜在的循环行为时触发拦截，防止因重放或循环导致的资源浪费和误执行。

Loop Guard 参数
LOOP_GUARD_TTL_MS
300000 (5 min)
循环检测窗口时长
LOOP_GUARD_ALLOW_COUNT
3
允许重复次数上限
Key 格式
sessionKey|runId|stableArgsKey
唯一标识一次循环上下文
监控的工具类型
write edit apply_patch exec bash memory_store
当同一 stableArgsKey 在 TTL 窗口内被调用超过 ALLOW_COUNT 次时，Loop Guard 触发拦截。此机制与 nonce 重放检测互补：nonce 检测完全相同的消息重放，Loop Guard 检测语义等价的循环行为。
