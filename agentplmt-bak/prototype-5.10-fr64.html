ClawManager 注册与认证流程
ClawManager API
ClawManager 负责 Agent 的完整生命周期管理。注册接口 POST /api/v1/agent/register 接受 Bearer agt_boot_* token 进行引导认证，验证通过后生成 agt_sess_* 会话令牌（24h 有效期）。InstanceAgent 数据模型包含 AgentID、SessionToken、SessionExpiresAt、Status（online/stale/offline）和 LastHeartbeatAt，session_token 字段设有 UNIQUE KEY 约束。
📦 注册阶段
1. Bootstrap Token 验证
• Agent 启动时携带预分配的 agt_boot_* token
• ClawManager 验证 token 有效性（未过期/未吊销）
• 验证 AgentID 与注册白名单匹配

2. Session Token 生成
• 生成唯一 agt_sess_* 令牌 (UNIQUE KEY)
• 设置 SessionExpiresAt = now + 24h
• Status 初始化为 online
• LastHeartbeatAt = now
💓 心跳与状态管理
Heartbeat 机制
• 间隔: 每 15s 发送一次心跳
• 滚动续期: 心跳成功时 SessionExpiresAt += 24h
• Stale 判定: >45s 无心跳
• Offline 判定: >120s 无心跳

状态流转
• online → stale: 45s 无心跳
• stale → offline: 120s 无心跳
• online/stale → revoked: 吊销操作触发
KubeArmor PKI 证书管理体系
KubeArmor TLS
KubeArmor 采用完整的 PKI 体系保障 mTLS 通信安全。CA 证书使用 RSA 4096 位密钥，有效期 3 年。系统维护 3 个 K8s Secret：kubearmor-ca、kubearmor-client、kubearmor-relay-server。TLS 模式使用 RequireAndVerifyClientCert 强制双向认证，CertLoader 支持 SelfSigned/External/K8s 三种证书加载模式，并支持证书轮换。

⚠ KSecure gRPC 安全提醒
KSecure 当前通过 PolicyService + LogService 提供 gRPC 服务，但 尚未启用 TLS。建议立即添加 mTLS 配置，与 KubeArmor PKI 体系对齐，确保策略下发和日志采集的通信安全。此为 AC-64.4 证书吊销机制的依赖项。
系统集成架构 — ClawManager + KSecure + KubeArmor
整体信任链建立在三个系统的协同之上：ClawManager 负责 Agent 注册、心跳监控和会话管理，是 Agent 身份的第一验证点；KubeArmor 负责 mTLS 证书签发和验证，在传输层确保通信双方身份可信；KSecure 负责安全策略下发和日志采集，是行为监控的策略引擎。当检测到 Agent 异常行为（由 FR-60/62 检测基线偏离和漂移）或越界操作（由 FR-61 拦截）时，FR-63 触发熔断，FR-64 执行吊销，形成从检测到响应的完整闭环。

Revocation Mechanism — Session / Certificate / Full
三级吊销机制

系统支持三种粒度的吊销操作，从最小影响的会话吊销到完全身份失效的全量吊销，可根据异常严重程度和风险等级选择合适的吊销策略。每种吊销类型均产生审计记录，确保操作可追溯。

Session Revocation 中级影响
会话吊销
机制: 将 agt_sess_* 会话令牌标记为已吊销，ClawManager 立即拒绝该 session token 的所有 API 请求（返回 401 Unauthorized）。
效果: Agent 无法继续通过 ClawManager API 接收命令或上报状态，但 mTLS 连接可能仍然有效（取决于证书状态）。
适用场景: Agent 行为异常但尚未确认恶意，需要暂停其操作进行审查；Agent 心跳异常但可能是网络抖动导致的临时问题。
恢复: 可通过重新颁发 session token 恢复（无需新 bootstrap token），但需要管理员审批。
Certificate Revocation 高级影响
证书吊销
机制: 将 Agent 的 KubeArmor client 证书加入 CRL（Certificate Revocation List）并更新 OCSP 响应，KubeArmor mTLS 在握手阶段验证证书状态时拒绝已吊销证书。
效果: Agent 的所有 mTLS 连接被拒绝，无法与 KubeArmor Relay、KSecure gRPC 等服务通信。即使 session token 仍有效，Agent 也无法建立传输层连接。
适用场景: Agent 身份可能被冒用，或证书私钥可能泄露；Agent 行为确认为恶意，需要彻底切断通信。
恢复: 必须重新签发新证书（需要 KubeArmor CA 重新签发），旧证书永久失效。
Full Revocation 完全影响
全量吊销
机制: 同时执行会话吊销 + 证书吊销 + Bootstrap Token 失效化，并触发 NetworkPolicy 全出站拒绝隔离。
效果: Agent 的会话、证书、引导令牌全部失效，网络出站被完全隔离，无法与任何系统组件通信。相当于从信任体系中完全移除该 Agent。
适用场景: Agent 被确认为 Rogue Agent，行为构成安全威胁；Agent 持续触发 Loop Guard (FR-63) 或执行越界操作 (FR-61)。
恢复: 必须重新注册，需要全新的 bootstrap token + 人工审批 + 30 分钟冷却期。
