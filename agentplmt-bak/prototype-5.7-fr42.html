Downgrade Detection Rules
降级检测规则
Enforce 4 规则
DR-001
Critical 协议版本降级
AgentProtocol: v1 → 未知版本 (e.g. "v0", "v2-beta", "")
Agent 注册时声明的 protocol_version 不匹配 AgentProtocolVersionV1="v1"，注册请求被拒绝。检测到使用弱协议或未声明的兼容模式。攻击者可能试图利用旧版协议的已知漏洞或绕过新版协议的安全增强。
MITRE_T1195 NIST_SI-4 AIA07
Enforce 动作: 拒绝注册 + 告警
DR-002
High Schema 字段缺失/降级
KubeArmorPolicy: 必填字段缺失 | 枚举值越界 | 模式不匹配
KubeArmor CRD 的 OpenAPI v3 schema 校验失败。包括: ActionType 使用 Allow/Audit/Block 之外的值；severity 超出 1-10 范围；path 不匹配 ^\/+.*[^\/]$ 模式；capability 使用未注册的 Linux capability；protocol 使用未知协议名；syscall 使用未注册的系统调用名。攻击者可能试图利用宽松的 schema 定义注入恶意规则。
MITRE_T1565 NIST_AU-2
Enforce 动作: PolicyStatus=Invalid(5) + 告警
DR-003
High 能力指纹降级 (Capability Reduction)
Agent: capabilities=["cmd_exec","file_rw"] → 注册时能力列表缩减或变更
Agent 注册时声明的 capabilities 列表与历史注册记录不一致。能力缩减可能意味着 Agent 被降级运行，能力增加可能意味着权限扩张。两种情况均需记录并验证，与注册时的能力指纹进行比对。未声明的兼容模式或隐式能力继承将被拒绝。
MITRE_T1562 NIST_AC-6 ASI03
Observe 动作: 告警 + 记录 (待升级为 Enforce)
DR-004
Medium API 版本回退
ClawManager API: /api/v1/ → /api/v0/ 或无版本前缀
客户端请求使用旧版或未知 API 版本路径。当前 ClawManager 仅注册 /api/v1 路由，任何非 v1 前缀的请求将返回 404。未来引入 /api/v2 时，需确保旧版 v1 仍有明确废弃策略。未版本化的 API 请求应被识别并告警。
