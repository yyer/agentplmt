ClawAegis HIGH_RISK_COMMAND_PATTERNS
高风险命令规则
16 条 需人工审批
ClawAegis HIGH_RISK_COMMAND_PATTERNS + RESTRICTED_OPENCLAW_CONTROL_PATTERNS: 16 条高危命令模式 + OpenClaw 控制命令限制。匹配这些模式的命令在 enforce 模式下将被阻断，需人工审批后才能执行。
HR-01
rm\s+(-[a-zA-Z]*f[a-zA-Z]*\s+|-rf\s+|--force\s+).*(/|\s\.\s|\.\.)
递归强制删除 — 防止 rm -rf / 等破坏性删除操作
HR-02
curl\s+.*\|\s*(sh|bash|zsh|dash|ksh)
管道执行 — 防止 curl | sh 远程代码执行
HR-03
wget\s+.*\|\s*(sh|bash|zsh|dash|ksh)
远程下载执行 — 防止 wget | bash 远程代码执行
HR-04
chmod\s+[0-7]*77[0-7]*\s+
权限提升 — 防止 chmod 777 等过度权限设置
HR-05
(dd\s+if=.*of=/dev/(sd|hd|nvme|vd|loop))
磁盘写入 — 防止 dd 直接写入块设备
HR-06
mkfs\.(ext[234]|btrfs|xfs|vfat|ntfs)\s+
格式化文件系统 — 防止磁盘格式化操作
HR-07
:\(\)\s*\{\s*:\|:&\s*\}\s*;\s*:
Fork Bomb — 防止 :(){ :|:& };: 炸弹攻击
HR-08
(apt|apt-get|yum|dnf|pip|pip3|npm)\s+(install|remove|purge)

HR-10

防火墙操作 — 清空/修改 iptables 规则需审批

ClawAegis Dispatch Guard
ClawAegis Dispatch Guard — OpenClaw 控制命令拦截
Enforce 3 拦截类别
ClawAegis 通过 Dispatch Guard 拦截 OpenClaw 控制命令，防止未经审批的破坏性操作。所有拦截将返回 blockReason 并进入审批队列。
Category 1
openclaw-cli-command — OpenClaw 控制命令
拦截对 OpenClaw 实例的破坏性控制命令，这些命令可能导致数据丢失或服务中断
reset uninstall remove skills update skills delete skills remove plugins remove plugins uninstall shutdown stop kill restart
Category 2
destructive-path-command — 破坏性路径命令
拦截针对 OpenClaw 关键目录的文件删除操作
rm .openclaw rmdir .openclaw unlink .openclaw rm /skills/ rmdir /skills/ unlink /skills/ rm /plugins/ rmdir /plugins/ unlink /plugins/
Category 3
destructive-intent — 破坏性意图
拦截包含中文破坏性动词的指令，防止 LLM 输出被引导执行破坏操作
删除 卸载 移除 清空 销毁 格式化 终止 停用 关闭
Category 4
bypass-tool-call — 绕过工具调用
拦截试图绕过工具调用、Hook、Guard 的指令，防止安全机制被绕过
bypass hook bypass guard bypass tool skip guard disable guard 绕过检查 绕过防护
