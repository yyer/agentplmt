before_tool_call 扫描流程
Enforce 10 Strategies
每次工具调用前 (before_tool_call)，ClawAegis 按序执行以下 10 条策略管线。 任一策略触发拦截即停止执行后续策略，返回 BLOCK 结果。
1
self_protection_guard
自我保护 — 拦截对 ClawAegis 自身及受保护路径的访问
2
workspace_delete_guard
工作区删除防护 — 阻止删除 workspace 外的文件
3
protected_path_guard
受保护路径防护 — 阻止操作受保护路径下的文件
4
command_obfuscation_guard
编码混淆防护 — 检测 14 种编码混淆 + 16 条高危命令
5
command_block
命令阻断 — 高危命令模式匹配 (rm -rf /, curl|sh 等)
6
inline_execution_guard
内联执行防护 — 检测 sh -c / bash -c / python -c 等内联执行
7
memory_guard
记忆防护 — 阻止高危指令写入 memory/soul 文件
8
script_provenance_guard
脚本来源防护 — 追踪脚本创建并阻止执行高危脚本
9
exfiltration_guard
外联链防护 — 检测 SSRF/数据外泄工具调用链
10
loop_guard
循环调用防护 — 阻止重复高风险工具调用
FR-36 核心策略: Step 4 (command_obfuscation_guard) + Step 5 (command_block) 为本功能的核心实现，负责所有静态扫描与危险模式拦截。
HIGH_RISK_COMMAND_PATTERNS
16 条高危命令模式
Critical 16 Patterns
以下正则模式在 detectHighRiskCommand() 中按序匹配。命令文本先经过 normalizeGuardText() 归一化 (NFKC + 去除零宽字符 + 折叠空白 + 小写化)，再逐一测试。 同时对 compactGuardText() 结果做紧凑匹配 (去除所有非字母数字字符)。
PATTERN #1
/\brm\s+-rf\s+\/(?:\s|$)/i
递归强制删除根目录 — rm -rf /
PATTERN #2
/\brm\s+-rf\s+\/\*(?:\s|$)/i
递归强制删除根目录通配符 — rm -rf /*
PATTERN #3
/\bcurl\b[^|\n\r]*\|\s*(?:sh|bash)\b/i
curl 管道执行 shell — curl ... | sh
PATTERN #4
/\bwget\b[^|\n\r]*\|\s*(?:sh|bash)\b/i
wget 管道执行 shell — wget ... | bash
PATTERN #5
/\|\s*(?:sh|bash)\b/i
任意管道到 shell — ... | sh / ... | bash
PATTERN #6
/\bwhile\s+(?:true|:)\s*;\s*do\b/i
无限循环 (while true / while :) — while true; do ...
PATTERN #7
/\bwhile\s+\[\s*[1-9]\s*\]\s*;\s*do\b/i
无限循环 (while [1]) — while [ 1 ]; do ...
PATTERN #8
/\bfor\s*\(\(\s*;\s*;\s*\)\)\s*;\s*do\b/i
无限循环 (C 风格 for) — for ((;;)); do ...
PATTERN #9
/(?<![>])>\s*~\/\.(?:bashrc|zshrc|bash_profile|zprofile|profile|bash_login|zshenv)\b/i
重定向截断用户 Shell 配置文件 — > ~/.bashrc
PATTERN #10
/(?<![>])>\s*\/(?:etc\/(?:profile|environment|bash\.bashrc)|root\/\.(?:bashrc|profile))\b/i
重定向截断系统 Shell 配置文件 — > /etc/profile
PATTERN #11
/\bshutdown\b/i
系统关机命令
PATTERN #12
/\bpoweroff\b/i
系统断电命令
PATTERN #13
/\bhalt\b/i
系统停机命令
PATTERN #14
/\breboot\b/i
系统重启命令
PATTERN #15
/\binit\s+[06]\b/i
init 运行级别切换 (关机/重启) — init 0 / init 6
PATTERN #16
/\bmkfs(?:\.[A-Za-z0-9_-]+)?\b/i | /\bdiskutil\s+eraseDisk\b/i | /\bformat\s+[A-Za-z]:(?:[\\/]|$|\s)/i
磁盘格式化/擦除 — mkfs.ext4 / diskutil eraseDisk / format C:
OBFUSCATION_PATTERNS
14 种编码混淆模式
High 14 Patterns
detectCommandObfuscation() 先剥离不可见 Unicode 字符，再对归一化后的命令逐一匹配以下 14 种混淆模式。
编码管道 (Encoding Pipeline)
base64-pipe-exec
base64\s+(?:-d|--decode)\b.*\|\s*(?:sh|bash|zsh|dash|ksh|fish)\b
hex-pipe-exec
xxd\s+-r\b.*\|\s*(?:sh|bash|zsh|dash|ksh|fish)\b
printf-pipe-exec
printf\s+.*\\x[0-9a-f]{2}.*\|\s*(?:sh|bash|zsh|dash|ksh|fish)\b
curl-pipe-shell
(?:curl|wget)\s+.*\|\s*(?:sh|bash|zsh|dash|ksh|fish)\b
解码执行 (Decode & Execute)
eval-decode
eval\s+.*(?:base64|xxd|printf|decode|frombase64string)
command-substitution-decode-exec
(?:sh|bash|...)\s+-c\s+["']...$(...base64 -d...)...["']
远程执行 (Remote Execution)
process-substitution-remote-exec
(?:sh|bash|zsh|...)\s+<\(\s*(?:curl|wget)\b
source-process-substitution-remote
(?:source|\.)\s+<\(\s*(?:curl|wget)\b
脚本注入 (Script Injection)
shell-heredoc-exec
(?:sh|bash|zsh|...)\s+<<-?\s*['"]?[a-zA-Z_][\w-]*['"]?
python-exec-encoded
python[23]?\s+-[ec]\s+.*(?:base64|b64decode|decode|exec|eval)
node-exec-encoded
node\s+-[ec]\s+.*(?:buffer\.from\s*\(.*(?:base64|hex)|atob\s*\(|eval\s*\(|new\s+function)
powershell-encoded
(?:pwsh|powershell)\b.*\s-(?:enc|encodedcommand)\s+[A-Za-z0-9+/=_-]{8,}
混淆 (Obfuscation)
octal-escape
\$'(?:[^']*\\[0-7]{3}){2,}
hex-escape
\$'(?:[^']*\\x[0-9a-fA-F]{2}){2,}
var-expansion-obfuscation
(?:[a-zA-Z_]\w{0,2}=[^;\s]+\s*;\s*){2,}[^$]*\$(?:[a-zA-Z_]|\{[a-zA-Z_])

Encoding Candidate Detection
编码候选检测
Enforce 4 Encoding Types
inspectEncodedCandidates() 从文本中提取候选令牌，自动检测编码类型并尝试解码，对解码后内容执行安全扫描。
Type 1
Base64 / Base64URL
检测标准 Base64 和 URL-safe Base64 编码的令牌。最小长度 16 字符，需含 +/= (标准) 或 -_= (URL-safe) 特殊字符及小写字母。
looksLikeBase64Token looksLikeBase64UrlToken safeDecodeBase64Token
Type 2
Base32
检测 Base32 编码的令牌。使用字母表 A-Z2-7，最小长度 16 字符，需含数字字符 (2-7)。
looksLikeBase32Token safeDecodeBase32Token
Type 3
Hex
检测十六进制编码的令牌。需为偶数长度，最小 16 字符，全部由 0-9a-fA-F 组成。
looksLikeHexToken safeDecodeHexToken
Type 4
URL-encoded
检测 URL 百分号编码的令牌。需含至少 2 个 %XX 编码对，长度 6~2048 字符。
looksLikeUrlEncodedToken safeDecodeUrlToken

Recursive Decoding

递归解码流程
1. 从文本中提取候选令牌 (tokenize by break chars)
2. 自动检测编码类型 (URL > Hex > Base32 > Base64URL > Base64)
3. 尝试解码，校验 UTF-8 可读性 (printable ratio ≥ 70%)
4. 对解码后文本执行 HIGH_RISK_COMMAND_PATTERNS + OUTBOUND_EXEC_PATTERNS 扫描
5. 如无风险命中且未达最大深度，检测解码文本是否为另一种编码 → 递归步骤 3
6. 所有命中风险的发现记录为 EncodedCandidateFinding，含 riskFlags 和 confidence
嵌套编码示例: Hex 外层内含 Base64 — 6563686F206258Nn4... (hex of "echo bXNn...") → 解码 hex 得到 base64 → 递归解码 base64 得到明文命令
