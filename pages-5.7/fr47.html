Carrier Systems
承载系统架构：ClawManager + ClawAegis
ClawManager（命令下发 + K8s 管理）
InstanceCommand 类型:
• quarantine_skill — 隔离指定 Agent 技能
• stop_openclaw — 停止 OpenClaw 实例
• restart_openclaw — 重启 OpenClaw 实例
• handle_skill_risk — 处理技能风险
• disable_skill — 禁用技能

命令生命周期:
pending → dispatched → running → succeeded/failed

DesiredState:
• DesiredPowerState: running / stopped
• DesiredConfigRevisionID: 配置版本
• DesiredRuntimeAction: 运行时动作
• UpdatedBy: 操作发起者

K8s 能力:
• Pod 管理: start / stop / restart / delete
• NetworkPolicy: egress 限制
• 命令幂等: IdempotencyKey
• 超时控制: TimeoutSeconds
ClawAegis（Loop Guard + Dispatch Guard）
Loop Guard:
• 阈值: 3 次重复 / 5 分钟
• 动作: 自动阻断
• 阻断原因: "安全限制：检测到重复高风险工具调用，已停止本次操作。"

Dispatch Guard 拦截模式:
• openclaw-cli — CLI 直接调用
• destructive-path — 破坏性路径
• destructive-intent — 破坏性意图
• bypass-tool-call — 绕过工具调用

阻断原因常量:
• BLOCK_REASON_LOOP
• BLOCK_REASON_DISPATCH_GUARD
• BLOCK_REASON_EXFILTRATION_CHAIN

NetworkPolicy 限制:
• Egress 仅允许 DNS(53)
• 系统命名空间: 80, 443, 8443, 9001, 3128

Circuit Breaker Mode
自动熔断模式
Enforce（强制执行）
Observe（仅观察）
Off（关闭）
Enforce 模式：触发条件满足时自动执行熔断干预，包括隔离 Agent、停止实例、限制网络等操作。所有干预命令通过 InstanceCommand 下发，确保幂等性和可追溯性。
Observe 模式：仅记录告警和熔断事件，不执行实际干预动作。用于调优阈值和验证规则有效性。
Off 模式：关闭自动熔断，仅保留手动干预能力。仅建议在调试或维护期间使用。
RULE-001 | Loop Detection
循环调用检测
Critical Enforce
当 Agent 在短时间内重复调用同一高风险工具时，Loop Guard 自动检测并阻断。基于 ClawAegis 的 Loop Guard 机制，维护滑动时间窗口内的调用计数，超过阈值后触发 BLOCK_REASON_LOOP 阻断，并下发 quarantine_skill 命令隔离相关 Agent。
ClawAegis Loop Guard BLOCK_REASON_LOOP quarantine_skill sliding window
触发阈值: 3 次重复 / 5 分钟
冷却期: 10 分钟
默认动作: 自动阻断 + 隔离
阻断消息: "安全限制：检测到重复高风险工具调用，已停止本次操作。"
启用循环检测
自动检测并阻断重复高风险工具调用
RULE-002 | Fan-out Limit
扇出爆炸限制
High Enforce
当单个 Agent 或技能触发过多并发子链路（fan-out 超限），可能导致系统资源耗尽和级联故障。Fan-out 限制检测器追踪每个 Agent 的活跃子链路数，超过阈值后自动降级该 Agent，下发 disable_skill 命令并应用 NetworkPolicy 限制 egress 通信，仅保留 DNS 和系统命名空间端口。
ClawManager 编排层 disable_skill NetworkPolicy egress restriction
触发阈值: 最大 8 并发链路
冷却期: 15 分钟
默认动作: 降级 + 网络限制
恢复条件: 活跃链路降至阈值 50% 以下
启用扇出限制
限制单个 Agent 的并发子链路数量
RULE-003 | Error Rate Threshold
错误率阈值
High Observe
当 Agent 执行失败率在时间窗口内超过阈值时触发。统计每个 Agent 的 succeeded/failed InstanceCommand 比例，结合 ClawAegis 的 BLOCK_REASON_DISPATCH_GUARD 和 BLOCK_REASON_EXFILTRATION_CHAIN 事件，综合评估异常程度。超过阈值后下发 handle_skill_risk 命令进行风险处理。
InstanceCommand 统计 BLOCK_REASON_DISPATCH_GUARD handle_skill_risk error rate sliding window
触发阈值: 失败率 > 50% / 5 分钟
冷却期: 10 分钟
当前模式: Observe（仅告警）
建议动作: 降级 + 风险处理
启用错误率检测
监控 Agent 执行失败率并触发熔断
RULE-004 | Resource Exhaustion
资源耗尽检测
Medium Enforce
通过 K8s Metrics Server 监控 Pod 的 CPU 和 Memory 使用率。当任何 Agent Pod 的资源使用率持续超过阈值时，自动降级非关键 Agent，下发 stop_openclaw 停止实例，同时应用 NetworkPolicy 限制 egress 至仅 DNS(53) 和系统命名空间端口 (80, 443, 8443, 9001, 3128)，防止资源竞争导致的级联故障。
K8s Metrics stop_openclaw NetworkPolicy egress DNS:53 / 80,443,8443,9001,3128
触发阈值: CPU/Memory > 90% 持续 2 分钟
冷却期: 20 分钟
默认动作: 降级 + 网络限制 + Pod stop
恢复条件: 资源使用率降至 70% 以下
启用资源检测
监控 Pod 资源使用率并触发自动干预
RULE-005 | Deadlock Detection
死锁检测
Medium Enforce
检测 Agent 间因资源竞争或依赖等待导致的死锁状态。当两个或多个 Agent 互相等待对方释放资源超过阈值时间时，自动取消死锁链路。通过 ClawManager 调度器下发 stop_openclaw 命令停止死锁的 Agent 实例，必要时执行 Pod deletion 强制释放资源，并通过 restart_openclaw 恢复其中一个 Agent 进行重试。
ClawManager 调度器 stop_openclaw restart_openclaw Pod deletion
触发阈值: 互等待 > 30 秒
冷却期: 5 分钟
默认动作: 停止死锁 Agent + 重试
重试策略: 停止优先级低的一方
启用死锁检测
检测并自动解除 Agent 间死锁状态
RULE-006 | Dispatch Guard Anomaly
调度守卫异常检测
High Enforce
ClawAegis Dispatch Guard 拦截特定危险调用模式。当检测到 openclaw-cli（CLI 直接调用）、destructive-path（破坏性路径）、destructive-intent（破坏性意图）或 bypass-tool-call（绕过工具调用）模式时，自动阻断并下发干预命令。触发 BLOCK_REASON_DISPATCH_GUARD 或 BLOCK_REASON_EXFILTRATION_CHAIN 记录审计日志。
openclaw-cli destructive-path destructive-intent bypass-tool-call BLOCK_REASON_DISPATCH_GUARD BLOCK_REASON_EXFILTRATION_CHAIN
触发条件: 匹配任一拦截模式
冷却期: 无（即时阻断）
默认动作: 阻断 + 隔离 + 审计
通知: 实时告警运维人员
启用调度守卫
拦截危险调度模式并触发熔断
