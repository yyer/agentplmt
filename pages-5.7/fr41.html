FR-41 消息签名架构
多智能体消息签名与完整性保护流程
P0 关键

对代理消息、调度命令、注册描述、能力声明和上下文传递结果进行签名与完整性校验，防止中间人篡改、消息伪造和隐藏语义注入。承载系统为 ClawManager（命令签名 + 状态报告校验）+ KSecure/KubeArmor（数字签名与完整性校验）。

强制状态报告签名 (Enforce State Report Signature)
Agent 上报的 StateReport 必须包含 HMAC-SHA256 签名，验签失败丢弃状态更新。
强制配置校验和 (Enforce Config Checksum)

AgentRegister 必须使用 RSA-SHA256 签名 + Bootstrap Token，防止未授权 Agent 注册。
签名失败自动告警 (Alert on Signature Failure)
任何签名验证失败事件自动触发安全告警，告警包含消息类型、签名者ID、失败原因。

KubeArmor IMA 集成
内核级完整性度量架构 (IMA + BPF LSM)

KubeArmor 通过 BPF LSM hook bprm_check_security 拦截可执行文件加载，调用 bpf_ima_file_hash() 计算文件哈希，填充 HashContext 结构体进行进程级完整性校验。enableIMA 标志控制是否启用 IMA 哈希采集。

签名算法配置
数字签名与完整性校验算法
RSA-SHA256 使用 KubeArmor GenerateSelfSignedCert 生成的 RSA 2048 密钥对 (CertKeyPair: Crt *x509.Certificate + Key *rsa.PrivateKey)。ECDSA-P256 适用于对延迟敏感的消息通道。
HMAC-SHA256 使用 Agent 注册时分配的共享密钥，性能优于非对称签名。跨信任域场景建议使用 RSA-SHA256。
当前 ClawManager InstanceConfigRevision.Checksum 使用 SHA-256 格式 "sha256:<64-hex-chars>"。切换算法需同步更新 ClawManager Checksum 计算逻辑。
KubeArmor HashContext.HashAlgo 字段 (uint32) 控制哈希算法。SHA-256 (值=2) 为推荐配置。bpf_ima_file_hash() 函数支持此算法选择。
密钥轮转配置
签名密钥生命周期管理
RSA 2048 密钥对 (KubeArmor CertKeyPair) 的轮转周期。轮转时 GenerateSelfSignedCert 生成新密钥对和 x509 证书，旧密钥保留 grace period 用于验签过渡。
轮转后旧密钥仍可验签的过渡时间，确保在途消息不会因密钥轮转而验证失败。
Agent-Manager HMAC-SHA256 共享密钥的轮转周期。密钥通过注册时安全通道分发，轮转后需通知所有相关 Agent。
自动密钥轮转 (Auto Key Rotation)
到达轮转周期后自动生成新密钥对并平滑切换，无需人工干预。
轮转告警通知 (Rotation Alert)
密钥轮转时发送通知到安全运营团队，确保轮转事件可审计。
指纹验证配置
完整性指纹检查周期与策略
ClawAegis SelfIntegrityRecord 指纹重算和比对的间隔。频率越高检测越及时，但 CPU 开销越大。12 个文件 (SELF_INTEGRITY_FILES) 的 SHA-256 计算约耗时 5ms。
IMA 哈希通过 BPF LSM hook bprm_check_security 实时触发 bpf_ima_file_hash() 计算，无需轮询。轮询模式为降级方案。
Agent 接收 ConfigRevision 时立即验证 ContentJSON 的 SHA-256 与 Checksum 是否一致。"both" 模式提供双重保护，防止存储后篡改。
启用 IMA 集成 (enableIMA Flag)
KubeArmor enableIMA 标志。启用后 BPF LSM hook bprm_check_security 触发 bpf_ima_file_hash() 计算 HashContext。
篡改时自动告警 (Alert on Tampering)
SelfIntegrityRecord 指纹不匹配或 IMA HashContext 异常时自动触发告警。
篡改时自动隔离 (Auto Quarantine on Tampering)
检测到自完整性篡改时，自动触发 Agent 隔离 (quarantine_skill 命令) 并暂停相关会话。
