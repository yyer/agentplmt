Consensus Modes
三种共识模式
3 种模式
👥
多 Agent 复核 (N-of-M)
高风险决策需要 N 个独立 Agent 中的 M 个批准方可执行。每个复核 Agent 独立评估决策内容，避免群体思维。当 N-of-M 阈值满足时自动放行，否则进入下一级审批。
阈值: 2-of-3 (默认)
超时: 300s → 自动升级
适用: stop_openclaw, quarantine_skill, handle_skill_risk
⚖️
规则引擎否决
基于 ClawManager RiskRule 的自动规则检查。当 HighestAction 为 require_approval 或 block 时，规则引擎自动否决单 Agent 决策。支持正则匹配模式，自动推导 HighestSeverity 和 HighestAction。
规则: Pattern (regex)
动作: allow / route_secure_model / block / require_approval
Severities: low < medium < high
require_approval = block (等效)
👤
人工确认 (Human-in-the-Loop)
关键决策（如安全策略变更、金融操作、大规模批量操作）必须经过人工确认。系统在达到人工确认超时后，根据配置的回退动作处理（拒绝/升级/允许）。审批记录完整保留审计轨迹。
超时: 1800s (30min)
回退: reject (默认)
适用: 安全策略变更 / 金融操作 / 批量操作
System Integration
与现有系统集成关系
🏢 ClawManager 集成
✓
RiskRule 规则引擎
Pattern (regex) 匹配文本内容，Severity (low/medium/high) 推导最高风险，Action (allow/route_secure_model/block/require_approval) 决定处置方式。require_approval 自动触发共识流程。
✓
AI Gateway resolveTargetModel
非敏感→allow；require_approval 或 block→阻止请求；敏感且安全模型→allow；敏感且非安全→路由到安全模型或阻止。
✓
InstanceCommand IssuedBy
17 种命令类型（start/stop/restart_openclaw, quarantine_skill, handle_skill_risk 等），IssuedBy 字段追踪命令发起者，支持共识审批链路追踪。
🛡️ ClawAegis 集成
✓
Dispatch Guard (4 模式组)
检测破坏性命令（rm -rf /, mkfs, dd）和绕过尝试（base64, hex, xxd 编码混淆），拦截结果作为复核输入，增强共识决策的准确性。
✓
Loop Guard (循环检测)
检测重复工具调用和 Fork Bomb 等资源耗尽型攻击，当检测到循环行为时自动触发共识复核，防止单 Agent 陷入循环后扩散。
✓
Exfiltration Guard (4 链决策)
4 层链式决策机制，含 runtimeRiskFlags 运行时风险标记，检测 curl/wget/nc 等外泄行为，拦截结果作为共识复核的辅助判断依据。
Acceptance Criteria
验收标准
✓
AC-1: 高风险决策支持多 Agent 复核
高风险命令（stop_openclaw, quarantine_skill, handle_skill_risk）和 RiskRule require_approval 触发的决策，必须经过 N-of-M 多 Agent 复核流程。
✓
AC-2: 复核结果可覆盖单 Agent 决策
规则引擎否决（require_approval = block）、多 Agent 复核拒绝、人工否决均可覆盖单 Agent 的原始决策，防止单一 Agent 的错误被直接扩散为系统级行为。
✓
AC-3: 共识流程可审计
完整的审计轨迹：谁发起→谁复核→如何决定→何时执行。包含 InstanceCommand IssuedBy 追踪、RiskRule 匹配记录、复核审批记录、超时升级记录。

Consensus Rules
共识触发规则
CRITICAL 高风险命令
Enforce
涉及删除、停止、隔离等不可逆操作的 InstanceCommand 命令类型，需要多 Agent 复核确认后方可执行。基于 ClawManager InstanceCommand 的 17 种命令类型分类。
触发条件: command_type in [stop_openclaw, quarantine_skill, handle_skill_risk, uninstall_skill, remove_skill]
共识类型: 多 Agent 复核 (N-of-M)
所需审批: 2-of-3 Agent 批准
超时/回退: 300s / reject
stop_openclaw quarantine_skill handle_skill_risk uninstall_skill remove_skill
HIGH 敏感数据访问
Enforce
导出或读取敏感数据时，触发 ClawManager RiskRule 的 require_approval 动作，需要人工审批或规则引擎自动路由到安全模型。对应 AI Gateway 的 resolveTargetModel 决策路径。
触发条件: RiskRule HighestAction = require_approval
共识类型: 规则引擎否决 + 人工审批
所需审批: 1 位管理员确认
超时/回退: 600s / route_secure_model
require_approval route_secure_model IsSensitive=true !IsSecure
HIGH 大规模变更
Enforce
批量操作（如批量重启实例、批量应用配置修订）需要人工确认，防止单 Agent 的批量操作错误导致系统级故障。对应 InstanceCommand 的 apply_config_revision、reload_config 等批量操作命令。
触发条件: 批量操作 ≥ 3 个实例 或 apply_config_revision
共识类型: Human-in-the-Loop
所需审批: 1 位管理员手动确认
超时/回退: 1800s / reject
apply_config_revision reload_config batch ≥ 3 instances
CRITICAL 金融操作
Enforce
涉及支付、费用相关的操作需要双重审批（Dual Approval），确保至少两位授权人员确认后方可执行。对应 ClawManager 的 CostRecord 和 ModelInvocation 计费链路。
触发条件: 支付/退款/额度变更操作
共识类型: 双重审批 (Dual Approval)
所需审批: 2 位授权人员独立确认
超时/回退: 3600s / reject + 告警
payment refund quota_change CostRecord
HIGH 安全策略变更
Enforce
安全策略修改（RiskRule 新增/修改/删除、防护模块开关变更）需要复核确认，防止策略降级导致安全防护失效。对应 ClawManager RiskRule 的 CRUD 操作和 ClawAegis 防护模块配置变更。
触发条件: RiskRule CRUD / 防护模块配置变更
共识类型: 多 Agent 复核 + 人工确认
所需审批: 1-of-2 Agent + 1 位安全管理员
超时/回退: 900s / reject

按风险等级配置共识模式
LOW 低风险决策
低风险操作（如 health_check, collect_system_info）自动通过，无需共识复核。
MEDIUM 中风险决策
中风险操作（如 install_skill, update_skill）需要多 Agent 复核确认。
HIGH 高风险决策
高风险操作（如 stop_openclaw, quarantine_skill）需要多 Agent 复核 + 人工确认。
CRITICAL 关键决策
关键操作（金融、安全策略变更）需要多 Agent 复核 + 人工双重确认。
